## Executive Summary

This slice created the full Docker Compose v2-spec local development stack for the Hilo People product. Eight services were declared with real healthchecks and dependency chains: `postgres`, `redis` (Valkey), `litellm`, `minio`, `minio-init` (one-shot sidecar), `backend`, `worker`, and `frontend`.

One in-scope critical defect was found and fixed during verify-slice cycle 1: the LiteLLM healthcheck used `curl` which is absent from the official LiteLLM image (`ghcr.io/berriai/litellm:v1.83.14-stable.patch.3`). Debugger cycle 1/3 replaced the probe with a Python stdlib `urllib.request` one-liner — the only interpreter available inside the container. Cycle 2 verify confirmed LiteLLM reaches `(healthy)` at ~17s with `FailingStreak=0`.

**Deferred items (not defects):**
- R1: `docker compose build backend/worker` deferred until T003 full requirements.txt lands (P00-S01-T003).
- R5: `worker` `app.worker` module not created until P02-S04-T002; service declared, boot deferred.
- R6: `docker compose build frontend` deferred until T002 merges (SKIP_BUILD=1 escape hatch in Dockerfile).
- pgvector extension for postgres deferred to P01-S01-T001 (correct image then: `pgvector/pgvector:pg17`).

F2 (host port 5432 parametrization) was explicitly classified `scope_expansion` / `future_enhancement` and deferred by user decision — no FU created per human decision.

## Technical Decisions

| Decision | ADR source | Reference |
|----------|-----------|-----------|
| No `version:` key in docker-compose.yml | Compose v2-spec migration guide | Task pack §F, §G |
| `redis` service wraps `valkey/valkey:8-alpine`; DNS name `redis` preserved | Valkey 8 is Redis-OSS 7.2.4 compatible | Task pack §C; researcher note item 2 |

| LiteLLM healthcheck: `python -c 'urllib.request...'` instead of `curl` | `curl`/`wget` absent in official LiteLLM image | Debugger cycle 1/3 — `evidence/debugger/01-litellm-healthy-after-fix.out` |
| `ARG SKIP_BUILD=0` in frontend/Dockerfile | Guards against build failure until T002 lands | Task pack §C R6 |
| `python:3.12-slim-bookworm` for backend (not alpine) | Better wheel coverage for psycopg/numpy/torch | Researcher note item 7 |
| `nginx:stable-alpine` in frontend/Dockerfile | Tracks 1.30.x patch line; SPA serving has no schema coupling | Researcher note item 8, reconciliation section |
| NERDCTL CAVEAT comment block in docker-compose.yml | nerdctl v2.2.x silently ignores healthcheck/condition | Researcher note item 5, issue #2386 |
