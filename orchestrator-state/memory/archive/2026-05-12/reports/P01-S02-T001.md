## Open Items

### Informational (non-blocking, not requiring follow-up from this slice)

1. **Pydantic 422 structural envelope inconsistency**: When a field is missing or has the wrong type, FastAPI's `RequestValidationError` handler returns `{detail:[...]}` instead of the project `{data, meta, errors}` envelope. Custom field validators (e.g. password policy) DO produce the project envelope with `AUTH_SIGNUP_INVALID_PAYLOAD`. A platform-level `RequestValidationError` handler would unify both paths — out of scope for T001 (no acceptance row, no write_set coverage for a platform handler). Out-of-scope follow-up candidate at main-orchestrator discretion.

2. **`scripts/dev-restart.profile.sh` path bug**: invokes the verification_data bootstrap with `--source data/verification` from CWD `backend/`, which fails. Manual workaround: `--source ../data/verification`. Out-of-scope for T001; existing operational gap in dev tooling.

3. **`router.py` at 309 lines**: 9 lines over the ~300 cap. Single responsibility (sign-up presentation). Recommend extracting `_engine`/`_SessionLocal`/`get_db_session` to `app/db/session.py` in the next auth slice (T002), since those helpers will be reused across T002/T005. Validator-approved nit.

4. **R1-T001-S02**: `test_downgrade_removes_all_tables` drops all 9 tables. After full test suite, `alembic upgrade head` must be re-run to restore schema. Known test ordering gotcha — documented in PROGRESS.md.
