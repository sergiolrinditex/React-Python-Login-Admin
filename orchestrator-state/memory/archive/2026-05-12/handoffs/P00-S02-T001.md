### Important decisions (+ doc source reference)

1. **No `version:` key** in docker-compose.yml — Compose v2 spec; deprecated key removed. Rancher Desktop (moby + containerd) compatible. Source: task pack §F, §G.

2. **Service `redis` wraps `valkey/valkey:8-alpine`** — Service name is `redis` so `REDIS_URL=redis://redis:6379/0` resolves without DNS changes. Valkey 8 is Redis 7.x protocol-compatible. Source: task pack §C.

3. **`depends_on` with `condition: service_healthy`** on all dependency chains — prevents race conditions during `compose up`. backend → (postgres+redis+litellm healthy); worker → (postgres+redis healthy); minio-init → (minio healthy). Source: task pack §F, constraint R4.

4. **`worker` service with `restart: on-failure`** — `app.worker` module does not exist until P02-S04-T002. Service declared but will exit-fail until that slice. "worker boot" acceptance met at SERVICE DECLARATION level per task pack §C note 3 and risk R5.

5. **Frontend Dockerfile `ARG SKIP_BUILD=0`** — Guards against build failure when T002 has not merged yet (no package-lock.json, vite.config.ts). When SKIP_BUILD=1, creates minimal placeholder dist. Source: task pack §C (R6), §D.

6. **Backend Dockerfile uses `python:3.12-slim-bookworm`** (not alpine) — Avoids wheel build failures for psycopg/numpy/torch. Non-root user `appuser:appgroup` (uid/gid 1001) for security compliance. Source: task pack §C, §01-non-negotiables.md §Security.

7. **`.env.example` extended** — Added `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `MINIO_ROOT_USER`, `MINIO_ROOT_PASSWORD`, `LITELLM_MASTER_KEY`. All new keys are docker-compose service requirements discovered in this slice. Source: task pack §D extensions table; TECHNICAL_GUIDE §11.1. `POSTGRES_USER/PASSWORD/DB` values mirror the existing `DATABASE_URL` credential substring (`hilo:hilo@localhost:5432/hilo_dev`). Dev placeholders only.

8. **`scripts/minio-bootstrap.sh` as write_set extension** — minio-init entrypoint is long enough (>5 lines YAML) to warrant an external script. Follows BEFORE/AFTER/ERROR logging per non-negotiables. Source: task pack §D, §C minio-init note, §01-non-negotiables.md §Logging.

9. **`frontend/nginx.conf` as write_set extension** — nginx runtime stage requires a server block with `try_files $uri /index.html` for React Router SPA. Minimal security headers; full hardening deferred to P04. Source: task pack §D.

10. **`.dockerignore` as write_set extension** — Reduces build context and prevents `.git`, `orchestrator-state/`, `node_modules`, `.venv`, `__pycache__` from reaching the Docker daemon. Standard best practice.
