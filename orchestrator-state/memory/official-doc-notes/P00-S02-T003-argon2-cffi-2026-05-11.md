## Discrepancy

Task pack says `~23.x` but latest stable is `25.1.0`. The version gap is 2 major versions (23 → 25).

This is purely a version staleness in the task pack's suggestion — not a functional discrepancy. The API is identical:
- `PasswordHasher.hash(password)` → str
- `PasswordHasher.verify(hash, password)` → True (raises on mismatch)
- `PasswordHasher.check_needs_rehash(hash)` → bool (no `verify_and_update` method)

**Exceptions raised by `verify()`:**
- `argon2.exceptions.VerifyMismatchError` — password does not match hash (subclass of VerificationError)
- `argon2.exceptions.VerificationError` — other verification failure
- `argon2.exceptions.InvalidHashError` — hash format invalid

**No `verify_and_update()` method exists.** The correct idempotency pattern (task pack §N.1) is:
```python
ph = PasswordHasher()
try:
    ph.verify(stored_hash, plain_password)
    if ph.check_needs_rehash(stored_hash):
        new_hash = ph.hash(plain_password)
        # update stored_hash
except VerifyMismatchError:
    # password changed — re-hash and update
    new_hash = ph.hash(plain_password)
```

The import remains `from argon2 import PasswordHasher` and `from argon2.exceptions import VerifyMismatchError, VerificationError, InvalidHashError` — unchanged from ~23.x.
