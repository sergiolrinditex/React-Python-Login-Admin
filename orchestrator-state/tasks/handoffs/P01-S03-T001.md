## Architecture decisions

### Clean Architecture mapping
- `domain/` — pure TypeScript, zero external imports. IAuthRepository port decouples presentation from data.
- `data/` — HTTP adapters: AuthRepository implements IAuthRepository. httpClient.ts is a standalone utility.
- `presentation/` — React layer: AuthProvider, useAuth(), RequireAuth, RequireRole. Depends on domain ports.
- Composition root: AuthProvider wired inside `router.tsx` (NOT in providers.tsx) per P-7 memory pattern.

### Single-flight 401 refresh design
- Module-level `_inflight: Promise<string> | null` in httpClient.ts.
- When `_inflight !== null`, concurrent 401-triggered callers await the same promise (no new refresh call).
- After resolution: `_inflight = null` (reset in `finally` block for safety).
- `__authNoRetry` option flag prevents /auth/refresh and /auth/logout from triggering retry loop.

### Redirect-state pattern
- RequireAuth builds `?next=<current_path>` from `useLocation().pathname + search`.
- The `?next=` value is sanitized by `getSafeRedirect()` before embedding.
- SignInPage (P03-S01-T001) will read `?next=` via `useSearchParams()` and call `getSafeRedirect()` again before `navigate()`.

### Open-redirect guard (getSafeRedirect)
Rules applied (in order):
1. Reject null/undefined/non-string.
2. Must start with "/" (reject absolute URLs, paths without leading slash).
3. Must NOT start with "//" (protocol-relative external URL).
4. Must NOT contain "://" (absolute URL smuggled after "/").
5. Must NOT start with "\\" (Windows path trick).
6. Must NOT contain "\\" anywhere.
7. Must NOT start with "javascript:" or "data:" (toLowerCase check).
Default safe fallback: "/chat".

### Token storage invariant
- `accessTokenStore.ts`: module-level `let _token: string | null = null`. Only get/set/clear functions exported.
- NEVER `localStorage.setItem`, `sessionStorage.setItem`, `IndexedDB`, or `document.cookie` write.
- Token lost on page reload → AuthProvider's mount-time `/refresh` call rehydrates from HttpOnly refresh cookie.
- T01 verifies: Storage.prototype.setItem NOT called after set/clear operations.
