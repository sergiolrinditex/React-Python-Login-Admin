**Researcher decisions incorporated (from official-doc-notes RESOLVED):**
- Q1: pyotp==2.9.0 confirmed — no newer 2.x version, no 3.x.
- Q2: `verify(otp: str, for_time=None, valid_window=0) -> bool` — never raises on malformed input; constant-time internally.
- Q3: Pass `code` (str) directly — `isdigit()` is acceptable (no Unicode bypass risk for security); researcher recommended `re.fullmatch(r'\d{6}', v)` for Unicode hardening (non-blocking; implemented or noted per worktree decision).
- Q4: Plain base32 str; auto-padding; `binascii.Error` on invalid base32 — implementation wraps in try/except → `MfaSecretMissingError`.
- Q5: No CVEs against pyotp ≤2.x as of 2026-05-12.
