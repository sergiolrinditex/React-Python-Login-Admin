### Important decisions

- **D-SVC-409 (U.1a)**: Chose U.1a (catch IntegrityError in service.py, raise ModelDefaultConflictError). Clean Architecture: repository raises IntegrityError → service translates to domain error → router maps to HTTP. Mirrors `app/auth/repository.py:131` precedent for EmailAlreadyExistsError.
- **`_is_default_conflict()` helper**: Inspects `exc.orig.diag.constraint_name` (psycopg2) with str(orig) fallback for psycopg3 compatibility. Only translates the specific `ai_models_default_per_type_uidx` constraint (pgcode=23505). Other IntegrityErrors are NOT swallowed (re-raised).
- **D-ORM-INDEX (§I.2)**: Applied. `AiModel.__table_args__` now declares the partial unique Index. Prevents future `alembic --autogenerate` from proposing DROP INDEX (R5 mitigation from task pack §I.5).
- **T26 mechanism (U.2a)**: `concurrent.futures.ThreadPoolExecutor` with 2 workers + `threading.Barrier(2)` for synchronization. Both threads call TestClient.patch() simultaneously. The DB partial unique index is the enforcement mechanism (not app-layer SELECT-before-write).
- **Why T26 proves DB-level enforcement**: Both threads may pass `find_previous_default_id` SELECT before either commits. The app-layer `UPDATE … SET is_default=false WHERE id != target` in transaction A does NOT see transaction B's model (uncommitted). When transaction B commits after A, the partial unique index detects two `is_default=true` rows for the same `model_type` and raises UniqueViolation. The test explicitly verifies: exactly 1×200 + 1×409 + DB has exactly 1 default row.
- **audit.write_model_audit extended**: Added `failure_reason: str | None = None` parameter. Conflict path writes `metadata.reason='default_conflict'`. Non-breaking change (existing callers don't pass `failure_reason`, so it defaults to None).
- **TECHNICAL_GUIDE §10.3 / §6.2 amendment**: Source-of-truth docs NOT edited during active TASK_ID (per `05-runtime-write-contract.md §Source-of-truth edits`). Amendment needed: §10.3 should declare the partial unique index, §6.2 PATCH row should add 409 to error column. Registered as §M.4 FU candidate (out-of-scope doc-sync, non-blocking).
