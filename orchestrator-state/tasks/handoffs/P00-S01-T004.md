### Important decisions (with doc source references)

1. **Path A chosen** (bootstrap-completion files included) — Justified by: (a) PROGRESS.md "Known Issues" explicitly says `npm run build will fail until tsconfig.json is added (expected — T004)`; (b) T002 handoff defers tsconfig to T004 twice; (c) CHECKLIST T004 verify command is `npm --prefix frontend run build` which cannot pass without tsconfig+vite+index+main. See task pack §6.2/§6.3.

2. **tsconfig.node.json: added `"composite": true`** — Required for `tsc -b` project references. The previous version had `"noEmit": true` which is incompatible with `composite`. Source: TypeScript error TS6306/TS6310 at build time.

3. **`vite-env.d.ts` added** — CSS side-effect imports (`import './tokens.css'`) cause TS2882 without `/// <reference types="vite/client" />`. Standard Vite pattern.

4. **Vitest setup: `@testing-library/jest-dom/vitest`** — Per official-doc-notes/P00-S01-T004-testing-library-vitest-setup.md. NOT the legacy `/extend-expect` form.

5. **`afterEach(cleanup)` in setup.ts** — `@testing-library/react` auto-cleanup only fires when `globals: true`. This project uses `globals: false`, so explicit cleanup is required to prevent DOM accumulation across tests in the same file.

6. **`toHaveStyle` vs `getAttribute('style').toContain()`** — jsdom does not resolve CSS custom properties in computed styles. Tests asserting token usage check the raw `style` attribute string instead of computed style. This is the correct pattern for token verification in a jsdom environment.

7. **MobileFrame naming** — Used `MobileFrame` per TECHNICAL_GUIDE §7; instrucciones §7 calls it `MobileShell`. Documented in component docstring. See also task pack §3.1.

8. **TrackedLabel accepts `style` prop** — Showcase needed to pass layout-scoped inline styles for section headers and form demos. Added `style?: CSSProperties` to `TrackedLabelProps` with spread into base style. Validated that this does not override token values.

9. **react-router-dom used for main.tsx** — Used `createBrowserRouter` + `RouterProvider` from `react-router-dom` (the installed package). The researcher note mentions `react-router/dom` but `react-router-dom` v7.15.0 is the thin re-export and is the installed package per package.json. Both import paths work in v7.

10. **providers.tsx: `type` keyword on ReactNode import** — `verbatimModuleSyntax: true` requires type-only imports to use `import type` or `import { type X }`. Fixed `import { ReactNode, useState }` → `import { type ReactNode, useState }`.
