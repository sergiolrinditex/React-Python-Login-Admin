### Important decisions

1. **JWT shape**: `jwt_secret` (single HMAC `SecretStr`) was split into `jwt_private_key` + `jwt_public_key` (both `SecretStr`) per §10.2 + §11.1 RS256 contract. No startup validator added — deferred to P01-S02-T001. Rationale: adding a startup validator here would block the dev stack before keypairs are provisioned.

2. **jwt_public_key as SecretStr**: The public key is not technically a secret, but keeping it as `SecretStr` is the least-surprise default that prevents log leakage by default. Consuming code (P01-S02-T001) will call `.get_secret_value()` to get the raw PEM for RS256 verify. Trade-off documented in config.py docstring.

3. **S3_BUCKET_DOCUMENTS default value preserved**: §11.1 dev cell says `hilo-docs-dev`; current default was `hilopeople-rag-documents-dev`. Decision: kept current default (infra bucket naming is an ops concern, not a §11.1 constraint). Documented in config.py comment.

4. **max_upload_mb default 50 → 25**: §11.1 dev/staging/prod = 25. This is a policy change (no consumers yet — RAG upload lands in P02-S06). Safe to apply now.

5. **mcp_allowlist_domains default "" → "localhost"**: §11.1 dev = localhost. No consumers yet (MCP lands in P02-S07). Widening dev allowlist; safe.

6. **docker-compose.yml `replace_all=true`**: Both occurrences of `LITELLM_PROXY_BASE_URL` in compose (lines 209 + 269) were replaced atomically. Verified 2 occurrences of `LITELLM_BASE_URL:` after edit.

7. **_REDACTED_KEYS update**: Removed `jwt_secret` (orphaned — would never match `jwt_private_key` or `jwt_public_key`), removed `provider_encryption_key` (orphaned — never matches `encryption_key`). Added `jwt_private_key`, `jwt_public_key`, `encryption_key` explicitly. The broader `secret` + `key` substring patterns in the redaction processor would also catch these by substring, but explicit entries are defense-in-depth.

8. **git stash incident**: During baseline ruff check, `git stash` was run to verify pre-existing errors. The stash pop failed due to a conflict on `orchestrator-state/tasks/ledger.jsonl`. Used `git stash pop` failure recovery — manually re-applied all file writes using Write tool (re-wrote config.py, logging.py, .env.example; re-applied docker-compose.yml edit). All 4 files verified clean after re-apply.
