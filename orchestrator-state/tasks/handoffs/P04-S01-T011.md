### DIAGNOSIS

The root cause is cross-module lru_cache poisoning between test_admin_ai.py and test_mcp_registry.py:

1. At pytest collection time (alphabetical): `test_admin_ai.py` loads → sets `ENCRYPTION_KEY=KEY_A` (random). `test_mcp_registry.py` loads → sets `ENCRYPTION_KEY=KEY_B` (random).
2. At test execution time: `test_admin_ai.py` tests run; each function-scoped autouse teardown (L335) calls `reset_fernet_cache()` then `os.environ["ENCRYPTION_KEY"] = KEY_A`. This fires AFTER module-level code.
3. After the last admin_ai test, `KEY_A` is in the env and the cache is cleared.
4. `test_mcp_registry.py::test_T06` runs. Without the fix, `_get_fernet()` reads `KEY_A` from env (because admin_ai teardown restored it), builds `Fernet(KEY_A)`, and encrypts the secret. The test verifies with `Fernet(KEY_B)` → `InvalidToken`.

### Fix applied

Two-layer defense:

1. **Module-level reset** (lines 87-99 in the fixed file): before `from app.main import app`, import `reset_fernet_cache` and call it immediately after setting `ENCRYPTION_KEY=_TEST_FERNET_KEY`. This prevents collection-time cache poisoning.

2. **Autouse per-test fixture** `_restore_fernet_key_per_test` (lines 220-248): function-scoped autouse fixture that runs BEFORE each test, sets `os.environ["ENCRYPTION_KEY"] = _TEST_FERNET_KEY` and calls `reset_fernet_cache()`. This prevents execution-time poisoning from `test_admin_ai.py`'s teardown, which fires after all admin_ai tests but before the mcp_registry tests start.

Pattern mirrors: `test_admin_ai.py:334-336` (teardown restore) and `test_chat_stream.py:124` (module-level reset).

### Important decisions
- `encryption.py` was intentionally NOT changed. The `@lru_cache(maxsize=1)` on `_get_fernet` is the production contract D-ENC1 and must remain. `reset_fernet_cache()` already exists as the public test helper.
- The fix is test-only: only `test_mcp_registry.py` was modified.
- The existing `test_T10` fixture (lines 493-533) has its own `reset_fernet_cache()` calls in a `try/finally` block and is not disturbed by the module-level reset.
