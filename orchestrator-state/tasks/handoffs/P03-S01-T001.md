## Decisions

### D-T001-CHALLENGE-TRANSPORT
React-router v7 `navigate('/auth/2fa', { state: { mfa_challenge_token, expires_in } })`.
Rejected: (a) sessionStorage — token survives hard reload unnecessarily; (b) querystring — token appears in URL/history/server logs.
Router state is cleared on hard-reload; TwoFactorPage (P03-S01-T005) handles the fallback (redirect back to /auth/sign-in if state is missing).

### D-T001-NEXT-FORWARD
On MFA branch: forward `?next=` to `/auth/2fa?next=<safe>` so deep link survives 2-step flow.
Non-MFA: navigate directly to `getSafeRedirect(rawNext)` (default `/chat`).

### D-T001-USERFETCH-ON-SUCCESS
On no-MFA success: `authRepository.signIn()` calls `setAccessToken(token)` then `this.fetchMe(token)` internally.
Returns `{ kind:'success', accessToken, user }` so `SignInPage.onSuccess` can call `signInAccepted(token, user)` atomically.
Rejected: defer fetchMe to AuthProvider — would require changing the signInAccepted contract, larger blast radius.

### D-T001-PASSWORD-MIN
`zod.string().min(1)` on sign-in — no creation policy enforcement (backend `SignInRequest` is min_length=1 by design).
This prevents blocking legacy passwords that don't meet sign-up policy. Asymmetry documented.

### D-T001-EMAIL-CORP
`zod.email()` only — no corporate-domain allowlist. Backend does not enforce domain restriction on sign-in.
Follow-up required if domain allowlist ever becomes a business rule (source-of-truth amendment needed).

### D-T001-LOCALE-AT-VIEW
Uses i18next detected language (fallback `es`). Account preference sync happens post-login on AccountPage (P03-S02-T004).
No cross-cutting language store change in this slice (YAGNI).

### D-T001-USESIGNIN
Hook extracted (`useSignIn.ts`) to (a) keep SignInPage ≤300 LoC self-contained, (b) isolate loading/error state for tests, (c) DRY for future pages if policy converges.

### D-T001-AGGREGATE-401
Single generic copy "Email o contraseña incorrectos." for all 401 failures.
Backend returns BYTE-IDENTICAL body for unknown-email and wrong-password (anti-enum, T002 policy).
Frontend copy must NOT differentiate.

### D-T001-NOMFA-COVERAGE
Option (c) chosen: backend integration tests for T002 cover both branches deterministically.
Frontend unit tests (useSignIn.test.tsx T03) cover no-MFA via fake repo.
/verify-slice human gate covers MFA branch end-to-end with real verification user.
A second verification user with MFA disabled was rejected (extends data contract, outside write_set).

### D-T001-FOCUS-MGMT
Simple disabled flag for rate-limited state (KISS). No timer countdown or automatic re-enable.
Focus on first invalid field is provided by react-hook-form mode:onTouched + browser native behavior.

### D-T001-ROUTER (§D-T001-ROUTER)
SignInStub function removed. SignInPage imported from `pages/auth/SignInPage`.
Default redirect / → /auth/sign-in (was /showcase — P03 intent per task pack §D-T001-ROUTER).
Catch-all *  → /auth/sign-in (was /showcase).

### D-T001-I18N (§D-T001-I18N)

Both must stay in sync (i18n/index.ts is the runtime source; JSON files are for reference/public access).
