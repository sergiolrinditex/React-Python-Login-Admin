## Summary

Replaced all 11 angle-bracket placeholders in `.env.example` that broke `bash source` due to bash interpreting `<` as stdin redirection. Used option (b) literal bash-safe values for 10 lines, and a specific already-recognized placeholder `dev-encryption-key-placeholder` for ENCRYPTION_KEY (line 93) to preserve the `ensure_encryption_key` matcher in `scripts/setup-from-scratch.sh`.

**Critical deviation from planner strategy (WRITE_SET_DRIFT: none — only strategy adjustment):**

The planner proposed option (a) single-quote `'<change-me>'` for ENCRYPTION_KEY (line 91). Testing revealed this is incorrect: `grep -E '^ENCRYPTION_KEY=' | cut -d= -f2-` reads the RAW file bytes and returns `'<change-me>'` (with quotes), NOT the shell-decoded `<change-me>`. The comparison `[ "$current_key" = "<change-me>" ]` therefore FAILS to detect the placeholder. The correct bash-safe option that preserves the matcher is `dev-encryption-key-placeholder`, which is already recognized at line 105: `|| [ "$current_key" = "dev-encryption-key-placeholder" ]`. This change requires no modification to `scripts/setup-from-scratch.sh`.
