### Important decisions (+ doc source reference)

1. **react-router-dom v7.15.0 pinned** (see official-doc-note `P00-S01-T002-react-router-v7-consolidation.md`):
   - In v7, `react-router-dom` is a thin re-export wrapper over `react-router`.
   - `BrowserRouter` still exists (library/declarative mode) and `createBrowserRouter` is recommended for data mode (loaders/actions).
   - P01-S03-T001 planner must choose the routing API intentionally. This note is the handoff signal.

2. **zod v4.4.3 (not v3)** (see official-doc-note `P00-S01-T002-zod-v4-hookform-resolvers.md`):
   - Zod v4 is the `latest` tag. v3 is in `next` tag (LTS). This project starts with v4.
   - Import pattern: `import { z } from 'zod'` (top-level re-exports v4). Do NOT use `'zod/v4'` sub-path unless mixing v3/v4.
   - Breaking changes from v3: `.merge()` deprecated → use `.extend()`; `.min(n, 'msg')` → `.min(n, { message: 'msg' })`; `z.coerce.*` input type is `unknown` not `string`.
   - Future form slices (P03-S01, etc.) must not copy-paste v3 patterns.

3. **@hookform/resolvers v5.2.2**: supports zod v4 automatically via the same `zodResolver`. Import: `import { zodResolver } from '@hookform/resolvers/zod'`.

4. **Testing libs moved to devDependencies**: npm placed `@testing-library/*` and `jsdom` in `dependencies` (because they were installed without `--save-dev`). They were manually moved to `devDependencies` in `package.json` per standard practice (test-only tools do not belong in production bundle). Justified: `frontend/package.json` is in the declared write set; changing a dep's category is not drift.

5. **Exact pins (no `^` or `~`)**: Per `01-non-negotiables.md §Dependencies` and `STACK_PROFILE.yaml`. All 12 new top-level deps are exact. The lockfile is authoritative.

6. **vitest.config.ts created** (planner-amended write set, §3.5 risk #6 of task pack): Without `environment: 'jsdom'`, Vitest 4 defaults to `node` and React DOM mounting throws `document is not defined`. Added minimal config; `globals: false` keeps explicit imports required.

7. **No tsconfig.json** (per task pack §3.5 risk #4): `npm run build` was NOT run as a verify gate — the build would fail without tsconfig. Verify gate is only `npm run test -- --run`, which passes (Vitest uses esbuild, no tsconfig required).

8. **AppProviders export shape locked**: `export function AppProviders({ children }: { children: ReactNode })`. Downstream slices (P01-S03-T001, P03-S01-T001, etc.) import this exact name. Do not rename.

9. **No logging in providers.tsx** (justified): Purely declarative component composition. Per `01-non-negotiables.md §Logging`, BEFORE/AFTER logs apply to functions with business logic. Providers gain logs when they add runtime behavior (auth interceptor, language detector callback, etc.).
