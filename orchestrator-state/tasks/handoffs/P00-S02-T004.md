### Important decisions

1. **show_locals=False chosen over custom processor** — task pack options A/B/C; Option C is simplest and most robust (no regex false-positive risk).
2. **_REDACTED_KEYS extended** — not replacing, only appending `pwd`, `dsn`, `database_url`, `connection_string` per acceptance criteria #1. Existing keys preserved.
3. **No changes to router.py call sites** — T002's conservative exc_info=False remains. The global fix is defense-in-depth for future code. Both are now safe independently.
4. **Test isolation via _save_logging_state()/_restore_logging_state()** — needed because configure_logging() adds a StreamHandler(sys.stdout). Without cleanup, the handler points to a closed capsys buffer after the test, causing "I/O operation on closed file" in subsequent tests. The solution: save handlers before test, close/remove added handlers in finally, restore originals.
5. **BEFORE/AFTER logging pattern** — added `print(..., file=sys.stderr)` for BEFORE (before structlog is active), structlog `_internal.debug(...)` for AFTER (after structlog bootstrapped). This follows the logging rule strictly.
