### Researcher informational findings verified

- **Citation rendering tolerates pre-chunk citation arrival**: Ō£ō. `useChatStream` collects citations into `citations[]` state via `onCitation` regardless of order; ConversationPage renders them with `CITATIONS_ROW_STYLE` block. The implementation is order-agnostic ŌĆö it works whether citations arrive before, during, or after chunks. ### Defects

**DEFECT-1 (in-scope, needs_debug):** `stream.ts` line 353-357 returns `{ ok: true, value: undefined }` when the ReadableStream read loop exits with `reader.done=true` but WITHOUT having received `event: done`. This path is reached whenever the SSE connection closes prematurely (LiteLLM drops after meta). `useChatStream` receives `result.ok === true`, logs only, and does NOT call `setPhase("error_network")`. The `onDone` handler was never called so `setPhase("completed")` is also never called. Result: phase stays `streaming` forever, composer stays disabled, `NetworkErrorView` is never shown. Fix: after the `while (true)` loop breaks via `done`, if `onDone` was never fired, return `{ ok: false, error: new ChatNetworkError("Stream closed without done event") }`.

**ENVIRONMENT-BLOCKER (not a code defect):** LiteLLM proxy not configured in this dev environment (`/ready` returns `litellm: unknown`). Streaming success path (chunk accumulation ŌåÆ done ŌåÆ assistant text ŌåÆ citations inline) cannot be observed. After DEFECT-1 is fixed, the streaming path will show NetworkErrorView (correct behavior for env-broken streaming). To verify the full success path, LiteLLM proxy must be configured with a real model.

### Scope classification for DEFECT-1

- In scope: `stream.ts` is in the Write Set (`frontend/src/features/chat/stream.ts`). The fix is a 3-line change (replace `return { ok: true }` at line 357 with a ChatNetworkError return when no done event arrived). No new route/endpoint/table/journey needed. Debugger ŌåÆ retest cycle applies.
- DEFECT-1 is NOT an out-of-scope FU. Do not create a follow-up. Mark `needs_debug` and run debugger.
