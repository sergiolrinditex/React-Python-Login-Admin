### Strategy applied
Dual fix with clear ownership:
1. **Cambio 1** (source of bug): Modified `reset_migrations` autouse fixture in
   `test_migrations_0001_auth.py` — teardown changed from `_run_alembic("downgrade", "base")`
   to `_run_alembic("upgrade", "head")`. The BEFORE step still does `downgrade base` (necessary
   for tests to start from a clean slate), but teardown now restores head for subsequent test files.
2. **Cambio 2** (defense-in-depth): Created `backend/tests/integration/conftest.py` with
   session-scoped autouse fixture `_integration_schema_head` that runs `alembic upgrade head`
   once at the start and once at the end of each full integration session (2 calls total per run,
   no per-test overhead). BEFORE/AFTER logging under `ENABLE_VERBOSE_LOGGING`.
3. **Cambio 3** (no change): `backend/tests/conftest.py` inspected and left intact — no functional
   change needed.

### Important decisions
- Teardown `upgrade head` (not `downgrade base`) because the tests within the file already do
  `_run_alembic("downgrade", "base")` in their own BEFORE setup, so each test still gets a clean
  slate. The teardown is purely about suite hygiene for the tests that follow.
- Session-scoped safety net chosen over function-scoped to avoid ~30% overhead per test (only 2
  alembic calls per session). Per task pack §R1.
- `backend/tests/test_migrations_0002_ai_chat_rag_mcp.py` (top-level, not in integration/) is
  explicitly OUT OF SCOPE per task pack and acceptance criterion. It does not affect
  `pytest backend/tests/integration -v`.
- Remaining 6 failures are all pre-existing (confirmed by BEFORE baseline):
  - `test_auth_logout.py` ×3 — pre-existing (were already FAILING before fix)
