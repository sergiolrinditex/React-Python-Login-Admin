## Important decisions
- **D1 — `_REFRESH_COOKIE_PATH` constant**: extracted to avoid drift between setter and clearer. DRY principle; single point of truth for the path value. Reference: task pack §I constraint "byte-identical attrs".
- **D2 — `https://testserver` in T15**: httpx with ASGITransport routes ASGI transport regardless of scheme, but ONLY sends Secure cookies when the scheme is https://. Using http://testserver would silently not send the cookie even if Path was correct. This is the authoritative check for RFC 6265 §5.4 Path + Secure attribute behavior simultaneously.
- **D3 — T15 design as regression guard**: T15 FAILS if `_REFRESH_COOKIE_PATH` reverts to `/auth`. This was confirmed by running T15 against the unfixed code (it returned 401 due to no_cookie). Test is not a placebo.
- **D4 — `01-non-negotiables.md` `Path=/auth` example**: This rule file shows `Path=/auth` as an example of the pattern (HttpOnly cookie with bounded path). It is not a contract — the contract is in TECHNICAL_GUIDE §10.2. The rule establishes the pattern; the slice fixes the concrete value. No modification to the rule file (write-scope guard blocks it; documented here as §K observation per task pack).
