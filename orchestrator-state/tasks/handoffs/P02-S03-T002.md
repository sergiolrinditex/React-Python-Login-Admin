### Important decisions (+ doc source references)
- **D-CHATSTREAM-WIRE**: Included streaming router in `chat/routers/__init__.py` aggregator — NOT `main.py`. P-22 pattern (admin aggregator precedent).
- **D-CHATSTREAM-SSE / Option B**: Used `StreamingResponse` with manual SSE byte framing via `sse.py`, NOT `EventSourceResponse` from `fastapi.sse`. Rationale: fastapi.sse requires ≥0.135.0 (not pinned), avoids adding conditional import. Wire format is identical. Documented in `sse.py` docstring D-SSE1/D-SSE2.
- **D-CHATSTREAM-PARTIAL**: On asyncio.CancelledError or LLM mid-stream error → persist partial content with token_count=None in a fresh background `_SessionLocal()` (§K-CHATSTREAM-BGSESSION). See `persistence.py::persist_partial_turn`.
- **D-CHATSTREAM-CITORDER**: Citations emitted before first chunk (sources shown immediately). See §H.
- **D-STREAM1**: `litellm.acompletion(stream=True, stream_options={"include_usage": True})` pattern confirmed by researcher M.1. Usage on last chunk (choices=[]). Guard: `if chunk.usage is not None`.
- **D-COST1**: `litellm.completion_cost(completion_response=chunk)` for estimated_cost. Fallback to `ai_models.pricing` JSONB if LiteLLM returns nothing.
- **D-CANCEL1**: `await _safe_aclose(response)` on asyncio.CancelledError or StopAsyncIteration to avoid HTTP/2 stream leaks (researcher M.2 confirmed `aclose()` pattern).
- **D-EMBED1**: `litellm.aembedding(model=..., input=[text])` → `result.data[0].embedding`. Dimension enforced client-side. Researcher M.3 confirmed adding `dimensions=1536` is correct.
- **§K-PROMPT-FILE**: SYSTEM_PROMPT_PEOPLE_ASSISTANT constant defined inline in `service.py`. YAGNI — deferred versioned file at `app/prompts/system/*.md` to future slice when ≥2 prompts exist (P-19 pattern).
- **assistant_content redaction**: Added `assistant_content` to the log redaction list (§I security checklist); only lengths/counts logged.
