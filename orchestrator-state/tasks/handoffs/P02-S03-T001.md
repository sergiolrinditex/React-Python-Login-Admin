### Important decisions

- **D-PAG1**: Cursor = base64url("<updated_at_iso>|<uuid>"), limit+1 algorithm, next_cursor from last returned row (not the discarded sentinel). index `conversations_user_updated_idx` covers the WHERE+ORDER clause.
- **D-TIT1**: initial_message present → title = first 60 chars (strip + "..." if > 60). Absent → title = "" (empty string). Frontend renders i18n placeholder for empty title.
- **D-LANG1**: explicit language in request body → use it. Absent → use `current_user.preferred_language`.
- **D-TX1**: POST atomicity: `create_conversation` flushes conv row first (to get id), then inserts message, then `UPDATE conversations SET updated_at=now()`. Router calls `session.commit()` after service returns. Avoids `session.begin()` (would fail with "transaction already begun" error since `get_db_session` already starts a transaction on first query).
- **D-RL1**: No rate limit on chat endpoints in T001. Business threshold not specified. Redis limiter from P02-S02-T001 available but not wired here. Documented for future hardening.
- **D-AUD1**: Chat CRUD is NOT an auditable action per §Security/Audit log list.
- **D-LOG1**: structlog BEFORE+AFTER per endpoint/service/repository with user_id_hash (SHA-256[:16] of UUID), request_id, route, latency_ms, status_code. Content NEVER logged.
- **D-option-b**: Used local `ChatResponseMeta` (option b from task pack §F.2) to avoid WRITE_SET_DRIFT on `app/auth/schemas.py`. ResponseMeta in auth module unchanged.
- **session.begin() gotcha**: `get_db_session` uses `autocommit=False` — the session has an implicit transaction started by the first query (from `get_current_user`). Calling `with session.begin()` in the router fails with "A transaction is already begun on this Session." Fix: call `session.commit()` directly after the service call, not `with session.begin()`.
- **conversation.user_id nullable**: The ORM has `nullable=True` for user_id (anonymous/system conversations). In this slice, current_user.id is always non-null (auth guard), but WHERE clause uses `==` not `is_()` per D-R3.
