### Â§D-RR2 â€” ES fallback delegation to caller

Decision: The retriever does NOT implement a fallback to `language="es"` when the query language yields 0 results. Fallback is the caller's (chat streaming P02-S03-T002) responsibility.

Rationale: Separation of concerns â€” the retriever's contract is stateless cosine search with explicit filters. Implicit fallback behavior would be invisible to callers, break A7 (unknown collection = empty, not fallback), and complicate observability. The chat streaming layer can detect `len(results) == 0` and retry with `language="es"` if needed. Documented per task pack Â§4 (regla 66 fallback note) and Â§9.3 U1.

### Â§D-RR3 â€” Embedding dimension validated at input, not at DB read time

Decision: Dimension check happens in `_validate_embedding()` on `query_embedding` (input), not when reading persisted embeddings.

Rationale: pgvector enforces dimension consistency at INSERT time (migration 0002 declared `VECTOR(1536)`). A retriever receiving the wrong query_embedding dim would cause a pgvector DB error without a clear error message. Validating early (before DB round-trip) provides `InvalidEmbeddingDimensionError.actual_dim` for diagnostic clarity. If an Admin AI model change alters the embedding model default (P02-S05-T001 allows this), the migration must update the `VECTOR(N)` column â€” the retriever constant `EXPECTED_EMBEDDING_DIM = 1536` would then require a source-of-truth amendment. Documented per task pack Â§9.3 U4.
