### Important decisions (+ doc source reference)

1. **Pure routing, no LLM**: `select_subagent()` uses `set(re.findall(r"\w+", msg.lower()))` intersection with `subagent_topics`. TECHNICAL_GUIDE §15 ADR-002 specifies "mayor solapamiento de palabras clave" — this is keyword overlap, not semantic similarity. D1 from task pack: stub deterministic, no LiteLLM call in T008.

2. **SubagentExecutor Protocol + StubExecutor**: Decouples routing from LLM. `StubExecutor` returns canned responses by agent name. P02-S08-T001 will provide `RealSubagentExecutor` using `deepagents.create_deep_agent`. No `import deepagents` in T008 (R-T008-6 mitigation).

3. **AgentSeed import for validation**: `load_specs_from_json()` uses `AgentSeed.model_validate()` from `app.seeds.schemas.mcp_agents` — DRY, avoids schema drift. The domain `AgentSpec` is a plain frozen dataclass (no Pydantic, no external deps in domain layer).

4. **Tie-break rule**: Equal-score subagents: prefer larger `len(subagent_topics)` (broader coverage). Secondary: alphabetical `name` ascending. Documented in `routing.py` docstring and tested explicitly in `test_routing.py::test_tiebreak_*`.

5. **Message not logged**: `message` raw is never logged (PII risk). `message_hash = sha256(message)[:12]` is used instead. `system_prompt` is also never logged (may contain sensitive instructions).

6. **mcp_server_name loaded but not resolved**: R-T008-5 — `AgentSpec.mcp_server_name` is populated from JSON but the routing layer ignores it. MCP tool injection is P02-S08-T001 scope.

7. **Mode A JSON loader**: `load_specs_from_json()` is the T008 loader. Mode B (SQL `SELECT * FROM agents`) will be added by P02-S08-T001 when the `agents` table migration lands. The interface (same `AgentSpec` output) is already stable.

8. **WRITE_SET_DRIFT**: `backend/tests/unit/agents/**` and `scripts/demo_supervisor_routing.py` are outside the canonical `write_set: backend/app/agents/**`. Justified as Path-A precedent (T003 added `tests/integration/`, T004 added `scripts/` equivalents). These paths don't conflict with any declared write_set of active tasks.
