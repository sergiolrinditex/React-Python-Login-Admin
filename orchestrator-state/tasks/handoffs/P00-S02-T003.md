## Actions performed

### Architectural decision — Opción C refinada

Tables don't exist yet (created by P01-S01-T001). Three options were considered:
- **Opción A** (fail if tables missing): would make bootstrap non-runnable in P00 → rejected.
- **Opción B** (create tables in bootstrap): duplicates migration responsibility → rejected.
- **Opción C adopted**: `inspect().has_table()` at runtime. Each group loader checks its table at call time. If the table is missing, it returns `LoadResult(status="deferred", reason="table missing")` and logs a WARN. Bootstrap exits 0. When P01-S01-T001 runs migrations, the same code activates automatically without any changes.

### Idempotency with Argon2id

Argon2 uses random salt so same plaintext hashes differently each call. Pattern:
- **INSERT path**: always hashes fresh (`ph.hash(plain_password)`)
- **UPDATE path**: calls `ph.verify(stored_hash, plain_password)` — only re-hashes if verification raises `VerifyMismatchError` (password changed) or `InvalidHashError` (hash corrupt/upgraded params)

This avoids unnecessary DB writes on repeated seed runs.
