### Key decisions
- **EmailStr removed**: replaced with `str` + field_validator regex — `email-validator` not in dep stack; project rule: no package for <20-line work.
- **_comment stripping in io.py**: JSON fixture files use `_comment` fields as human-readable documentation. Rather than removing them (losing readability) or relaxing `extra="forbid"` (losing schema safety), `io.py` strips all `_`-prefixed keys at top level before Pydantic validation.
- **Static TOTP**: `JBSWY3DPEHPK3PXP` used as deterministic base32 secret; `pyotp` not installed (and not needed — the fixture value is static for verification reproducibility).
- **Credential guard**: all provider/MCP credential values must start with `synthetic-` AND must not match real key patterns (sk-*, sk-ant-*, AIza*, Bearer *). Implemented in `admin_ai.py._require_synthetic_prefix` and imported in `mcp_agents.py`.
- **Exit codes**: 0 = all ok (or all tables skipped), 1 = fixture/schema error, 2 = bundle directory missing.
- **Table-tolerant**: tables missing in P00 → WARN log + add to skipped_tables + continue. Does not exit non-zero. This is the explicit P00 contract.
- **Checksum length**: initial fixture had 66-char SHA256 hex (typo); fixed to exactly 64 chars (RagDocumentSeed enforces min_length=64, max_length=64).
