## Scope

**Goal**: Fix `check_handoff_contract.py` false-negative in pr-flow when called
from a cwd that is not the workspace root and `CLAUDE_WORKTREE_ROOT` is not
exported (e.g. from `/tmp` with only `CLAUDE_ORCHESTRATOR_ROOT` set). Before
this fix, `EXIT=2 "missing handoff"` was returned even though the handoff
existed in the canonical root. After: `EXIT=0` with fallback to canonical.

**Files changed**:
- `.claude/bin/check_handoff_contract.py` — 229 LoC → 383 LoC (+154 LoC net)
  - New imports: `STATE_DIR_NAME`, `project_root` from `common`; `os`
  - New function `_vlog()` (verbose stderr logging guard)
  - New function `_resolve_handoff_path()` (workspace→canonical fallback)
  - `validate()` now calls `_resolve_handoff_path()` instead of `_handoff_path_from_workspace()`
  - `main()` body wrapped in `try/except Exception` with clean error exit
  - Extended file docstring with slice/phase/dependency info
- `.claude/bin/tests/test_handoff_contract_resolution.py` — NEW 248 LoC
  - 4 new tests for dual-root resolution (see §Tests)

**Files NOT changed** (per write_set and non-negotiables):
- `common.py` — not in write set; resolver is local to the CLI only
- `scripts/check-handoff-contract.sh` — wrapper is correct (`exec` propagates exit)

## Decisions

**D-T007-RESOLVER**: `_resolve_handoff_path(task_id)` implements workspace→canonical
fallback locally in `check_handoff_contract.py` rather than modifying `common.py`.
Rationale: `common.py` is outside the write set (`Conflict group:
infra:orchestrator-scripts`); any change there affects ALL consumers (hooks,
scripts, agents) and requires a scope expansion FU. A local fallback in the CLI
is the minimum necessary change per YAGNI.

**D-T007-FALLBACK-ORDER**: Candidate A (workspace via `common.handoff_path`) is
tried first because during an active pr-flow slice the workspace is authoritative.
Candidate B (canonical via `project_root() / STATE_DIR_NAME / "tasks/handoffs"`)
is tried only on A-miss. This matches the "workspace authoritative during slice,
canonical authoritative after merge" doctrine in `05-runtime-write-contract.md`.

**D-T007-CLEAN-ERROR-2**: `main()` wraps the body in `try/except Exception`
(NOT `BaseException` — `SystemExit`/`KeyboardInterrupt` propagate normally).
The catch converts any unexpected crash into `Handoff contract FAILED —
<task_id>\n- internal error: <type>: <msg>` on stderr + exit 2. Justified as
the CLI entry point whose contract is exit-code stability (non-negotiable for
hooks and closer pre-check).

**D-T007-VLOG**: Added `_vlog()` helper (ENABLE_VERBOSE_LOGGING guard) to provide
BEFORE/AFTER resolution trace in the new function, satisfying the logging
non-negotiable (rule 01) without introducing `logging` module overhead in a
simple CLI tool.

**D-T007-WRAPPER-NO-CHANGE**: `scripts/check-handoff-contract.sh` uses `exec
python3 -B -S "$ROOT/.claude/bin/check_handoff_contract.py" "$@"` which
propagates exit codes correctly. The duplicate FU-20260514045116 claim of
"wrapper masking" is factually incorrect against the current wrapper — verified
in smoke tests. Wrapper was NOT touched.
