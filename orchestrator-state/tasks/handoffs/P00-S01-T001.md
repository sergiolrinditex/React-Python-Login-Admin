### Important decisions

1. **`--check` mode added to `scripts/setup-from-scratch.sh`**: The script did not have a `--check` mode. When `backend/app` exists, the script tries to run `alembic upgrade head` which fails without deps installed. The `--check` mode skips all declared commands (DB migrate, DB seed) without failing, allowing structural validation only. This is the smallest possible change within `scripts/**` (allowed path). Reference: task pack `Riesgos y mitigaciones #1` + `scripts/**` in write set.

2. **`fastapi==0.115.12` + `uvicorn[standard]==0.34.2`**: Pinned to current stable releases at bootstrap time (2026-05). official-docs-researcher will verify; if versions are superseded, T003 can pin correctly.

3. **`FRONT_PORT=5173`** (not 5000): STACK_PROFILE `frontend.dev_cmd = npm run dev -- --host 0.0.0.0` combined with Vite default port 5173. `dev-restart.profile.sh` uses `FRONT_PORT` (default 5000 in that file's comments), but the `.env.example` uses 5173 matching Vite default so the frontend dev starts on the correct port out of the box. This aligns with `FRONTEND_PORT=5173` in the task pack impact analysis §6.

4. **`API_PORT` / `FRONT_PORT`** as variable names (not `BACKEND_PORT`/`FRONTEND_PORT`): These are the canonical names declared in `scripts/dev-restart.profile.sh`. Using the names that the actual profile reads from `.env`.

5. **No `backend/__init__.py` at repo root**: Task pack says "prefer NO root `__init__.py`". Only `backend/app/__init__.py` created (package marker so `uvicorn app.main:app` resolves).

6. **`main.py` stdlib logging**: `structlog` is installed in T003. For this slice, `logging.basicConfig` with `ENABLE_VERBOSE_LOGGING` flag respects the non-negotiable logging contract without introducing external deps.
