### Key decisions

- **D1 (email-validator dep)**: Pydantic v2 `EmailStr` requires `pip install 'pydantic[email]'` (email-validator>=2.0.0) — this is NOT a transitive dependency of pydantic-core. Pinned to `email-validator==2.2.0` per project pin-exact-versions policy. Marked official doc note RESOLVED.
- **D2 (T005 audit_log native columns)**: Migration 0002 (T005) already added ip, user_agent, request_id, resource as native nullable columns. Used them directly instead of JSONB stash. Compatible with both approaches.
- **D3 (mfa_required=True always, no TOTP insert)**: sign_up does NOT insert into mfa_totp_secrets — that requires an encrypted TOTP secret generated at enrollment time (P01-S02-T003 or similar). Per task-pack §9 D3.
- **D4 (409 non-leaky)**: Duplicate email returns `AUTH_EMAIL_TAKEN` + "Email no disponible. Por favor utiliza otro email." — does NOT say "already registered" or expose registration status. OWASP-compliant.
- **D5 (event loop isolation)**: pytest-asyncio 1.3.0 `asyncio_mode=auto` creates a new event loop per test function. asyncpg connections are bound to their creating event loop. The app's singleton engine (`app.core.db._engine`) was reused across tests with different loops → "Future attached to different loop" error. Fix: `reset_db_engine_singleton` autouse fixture resets the module-level singletons before each test, forcing a fresh engine in each test's loop.
- **D6 (ASGITransport pattern)**: `AsyncClient(transport=ASGITransport(app=app), base_url="http://test")` — the recommended non-deprecated pattern per httpx/starlette docs (not `AsyncClient(app=app)`).
- **D7 (corporate email permissive default)**: `CORPORATE_EMAIL_DOMAINS=""` → permissive (any domain accepted). Matches verification user `s.lopezrap+employee@gmail.com` domain. Production must set this.
