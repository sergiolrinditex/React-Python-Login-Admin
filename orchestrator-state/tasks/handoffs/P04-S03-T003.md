### Important decisions (+ doc source reference)

- **D-AUDIT-READONLY** (task pack §D-AUDIT-READONLY): `GET /api/v1/admin/audit` is read-only.
  `audit/audit.py` is a stub with `write_audit_query_audit()` no-op for structural symmetry with
  `providers/audit.py` and `model_catalog/audit.py`. No `audit_logs` row is written for this GET.
  Consistent with `admin/usage.py` precedent ("No audit row for GET /admin/usage").
- **D-AUDIT-MAIN** (task pack §D-AUDIT-MAIN-MOUNT): `main.py` registers `audit_router` with
  `prefix="/api/v1/admin"` so the route resolves as `GET /api/v1/admin/audit`. Annotated with
  `# P04-S03-T003 WRITE_SET_DRIFT §D-AUDIT-MAIN` comments at lines 52 and 99.
- **D-AUDIT-INIT** (task pack §D-AUDIT-PACKAGE-INIT): `__init__.py` is required for Python
  package resolution; declared as `WRITE_SET_DRIFT §D-AUDIT-INIT` in handoff (Coverage Registry
  write_set notation `{router,service,repository,schemas,audit}.py` is non-exhaustive; `__init__.py`
  is implicit per same precedent as D-AAP in P02-S05-T001).
- **D-SVC-WINDOW** (task pack §Unknowns item 1): `from_dt` + `to_dt` are REQUIRED; window must
  be > 0 and ≤ 90 days. Returns `AUDIT_WINDOW_TOO_WIDE` (422) if cap exceeded.
  Returns `AUDIT_WINDOW_INVALID` (422) if from >= to. Mirrors `admin/usage.py` precedent.
- **D-SCHEMA-METADATA** (task pack §Unknowns item 5): Pydantic field `metadata` maps to ORM
  attribute `extra_metadata` (DB column `metadata`). ORM uses `extra_metadata` to avoid
  SQLAlchemy `InstrumentedAttribute` clash with Python built-in `metadata`. API always exposes
  `metadata` for JSON consistency.
- **D-PERM1** (task pack §Auth contract): `require_auditor = require_role("people_auditor")` and
  `super_admin` is a superset. Tests T01 and T02 verify both paths.
- **D-REPO-LIMIT-PLUS-ONE** (task pack §Impact analysis / repository): fetches `limit+1` rows to
  detect `has_more=True` without a separate COUNT query.
- **D-REPO-CURSOR** (task pack §Impact analysis / repository): opaque base64 over
  `"created_at_iso|id_uuid"`. Cursor decode errors treated as invalid (graceful: restarts from
  top of window).
- **D-ROUTER-NO-RATE-LIMIT**: read-only admin GET, same precedent as usage.py and
  admin/providers/router.py::list_providers.
