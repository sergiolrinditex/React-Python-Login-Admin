### Decision alignment
- D1 (idempotency = aggregate 401 on every failure path, cookie cleared on every 401): `routers/logout.py:96-110` catches `SessionExpiredError` → calls `_error_response(...)` → `_clear_refresh_cookie(err_resp)` BEFORE return. 204 path at `132-138` also calls `_clear_refresh_cookie(resp)`. Bodies are byte-identical because `_error_response` uses fixed `code=AUTH_SESSION_EXPIRED` / `message=str(exc)` and exc is `SessionExpiredError()` with default constructor (no reason leak). T10 (`test_auth_logout.py:541-598`) asserts this across four representative branches with `meta.request_id` stripped.
- D2 (Bearer AND cookie BOTH required, mismatch → 401 `user_mismatch`): `LogoutUser.execute` validates Bearer first (`_validate_bearer`, lines 198-261 with 4 reason branches `no_bearer|invalid_bearer|expired_bearer|<generic>`), then cookie (`no_cookie` at 123-135, lookup at 137-160), then `user_id` match at 162-177. UUID comparison via `_safe_uuid(bearer.sub) == row.user_id` is correct (UUID type compare, not string compare — avoids prefix collision).
- D3 (single-session revoke only by token_hash): `self._repo.revoke(token_id)` at line 181 — passes only the single row id from `find_active_by_hash_for_update`. No "logout all" code path; no extra DELETE/UPDATE on `refresh_tokens` table elsewhere in the diff. T11 (`test_auth_logout.py:605-628`) asserts user's other active token row remains untouched.

### Byte-identical 401
- All branches raise `SessionExpiredError()` with no constructor args → same `exc.code = AUTH_SESSION_EXPIRED`, same `str(exc)` message. `_error_response` uses fixed `code` + `message` + `http_status=401` + only varying `request_id` in `meta`. T10 strips `meta.request_id` and asserts `parsed[i] == reference` across no-bearer / invalid-bearer / no-cookie / unknown-cookie. Reason never appears in response body — confirmed by reading `errors.py:SessionExpiredError` (no `details` attribute used here) and by every 401 unit test only asserting `errors[0].code`.

### Cookie clearing semantics
- `_clear_refresh_cookie` in `_helpers.py:96-120` uses `httponly=True, secure=True, samesite="lax", path="/auth", max_age=0` — byte-identical attrs to `_set_refresh_cookie` (same module, adjacent function). Without identical attrs the browser will not actually delete the cookie, so this matters. T01, T02, T05 all assert `max-age=0` in `Set-Cookie` for both 204 and 401 paths.
- One small note (not a finding): `_clear_refresh_cookie` is also called on the 500 `AUTH_LOGOUT_INTERNAL_ERROR` branch in `routers/logout.py:111-125`. That branch is defensive (catches non-`SessionExpiredError` exceptions) and uses a different error `code`, but it is NEVER triggered by the test suite — there is no `_error_response` envelope mismatch with T001-T003 contract because the contract only specifies the 401 envelope. Acceptable defensive code.

### Audit logging
