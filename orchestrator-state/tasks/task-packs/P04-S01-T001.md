**Effective backend shape (read from `backend/app/admin/usage.py` HEAD — authoritative):**

- Method/path: `GET /api/v1/admin/usage`.
- Query params:
  - `from` — datetime ISO 8601, REQUIRED. 422 if absent.
  - `to` — datetime ISO 8601, REQUIRED. 422 if absent.
  - `group_by` — `"model" | "day" | "model_day"`, default `"model"`.
  - `model_id` — optional UUID filter.
  - `provider_id` — optional UUID filter.
- Status codes:
  - 200 OK — happy path.
  - 401 `AUTH_SESSION_EXPIRED` (session invalid / not authenticated).
  - 403 — non-admin authenticated user.
  - 422 `ADMIN_USAGE_INVALID_PAYLOAD` — bad `group_by` or `from >= to`.
  - 422 `ADMIN_USAGE_WINDOW_TOO_WIDE` — window > 90 days.
  - 500 `INTERNAL_ERROR` — DB failure.
- Response body 200 (verbatim shape):

```jsonc
{
  "data": {
    "from": "2026-04-16T00:00:00+00:00",
    "to":   "2026-05-16T00:00:00+00:00",
    "group_by": "model",
    "rows": [
      {
        "tokens_in": 1234,
        "tokens_out": 567,
        "estimated_cost": 0.0123,
        "latency_ms_avg": 842,
        "invocations": 12,
        "model_name": "gpt-4o-mini"      // present when group_by in {model, model_day}
        // "day": "2026-05-15"           // present when group_by in {day, model_day}
      }
    ],
    "totals": {
      "tokens_in": 1234,
      "tokens_out": 567,
      "estimated_cost": 0.0123,
      "invocations": 12,
      "latency_ms_avg": 842
    }
  },
  "meta": { "request_id": "uuid-v4" }
}
```

**Empty-state body**: 200 OK with `data.rows = []` and `data.totals` all zeros — this is the "empty"
UX state (admin has no usage in the window), distinct from `permission_denied` and `error_network`.

## 3. Decisions (record canonical decisions here so developer doesn't re-debate them)

### D-T001-GUARD: use existing `RequireRole`, no new `RequireAdmin`

`features/auth/presentation/RequireRole.tsx` already accepts `roles={["people_admin",
"super_admin"]}` and is currently wrapping `/admin` in `router.tsx`. Creating a `RequireAdmin`
HOC would be:

- redundant (RequireRole is polymorphic by design),
- speculative (no second consumer needs it as a named alias),
- conflict-inducing (other admin pages P04-S01-T002..T004 + P04-S02-* will reuse the SAME
  `RequireRole` block via shared `<Route element={<RequireRole .../>}><Outlet/></RequireRole>`).

**Decision**: replace the inner `AdminStub` element with `<AdminDashboardPage />` inside the
existing RequireRole route block in `router.tsx`. Do NOT introduce `RequireAdmin`.

### D-T001-NAV-LINKS: enabled links to whole admin route inventory

The dashboard is a "Vista resumen Admin AI" (TECHNICAL_GUIDE §6.1 last column) with a left
nav (use `AdminShell` from design-system; the nav model is already a list of
`{key,label,onClick}`). Per the Navigation Contract §6.4, admin routes are 11 paths in total.

**Decision**: render all 11 admin nav links **enabled**. React Router will resolve unimplemented
routes through the catch-all → `/` → `/chat` (for an authed admin, that is `/chat`; not ideal,
but no broken-link error). When subsequent P04-S01-T002+ slices land, the navigation already
works without further edits to AdminDashboardPage. Future-proof, zero-speculative-code: nav
items are pure data, not new components.

Nav inventory in left-rail order (TECHNICAL_GUIDE §6.4):

1. `/admin` — Dashboard (active)
2. `/admin/ai/models` — Modelos
3. `/admin/ai/models/new` — Nuevo modelo
4. `/admin/rag/documents` — Documentos RAG
5. `/admin/rag/collections` — Colecciones
6. `/admin/ai/mcp` — Servidores MCP
7. `/admin/ai/mcp/new` — Nuevo MCP
8. `/admin/ai/agents` — Agentes
9. `/admin/audit` — Auditoría
10. `/admin/usage` — Coste y latencias
11. (optional) sign-out / back to `/chat`

`AdminShell` renders these; the active one is `/admin` for this page.

### D-T001-NEXT-ACTION: page-level CTA to `/admin/ai/models`

TECHNICAL_GUIDE §6.1 declares "Next action: /admin/ai/models". Render a primary CTA
(`SolidCTA`) in the dashboard main area labeled `admin-ai:dashboard.cta.manageModels`.

### D-T001-WINDOW-DEFAULT: client computes last 30 days

Backend requires `from`+`to`. Default window = 30 days ending at "now" (UTC). Surface the
window on screen (`from→to`). Out of scope: a date-range picker (will live in `UsagePage`
P04-S03-T002).

### D-T001-GROUPBY-DEFAULT: `group_by=model`

Matches backend default. The dashboard shows usage **by model** in a `HairlineTable`
(model_name, invocations, tokens_in, tokens_out, estimated_cost, latency_ms_avg). 6 columns
fit the 1440px AdminShell layout.

### D-T001-KPIS: top-line totals row + hairline-table rows

Above the table, render a 4-tile KPI row from `data.totals`:

- Invocaciones
- Tokens (in + out)
- Coste estimado
- Latencia media (ms)

Each tile is hairline-bordered, label is `TrackedLabel` uppercase, value is large serif via
`--font-display`. This satisfies "usage summary" from acceptance.

### D-T001-UX-STATES: 5 states (no error_validation)

Per UX_CONTRACT §`/admin` row + TECHNICAL_GUIDE §6.1 (NOT §3.2's feature-level list):

| State | Trigger | Render |
|---|---|---|
| `loading` | `useQuery.isLoading` | `aria-busy` shell with skeleton hairlines (no spinner) |
| `empty` | 200 + `rows.length === 0` && `totals.invocations === 0` | Wordmark + body + manageModels CTA |
| `error_network` | network failure or 5xx | error view + retry CTA (refetch) |
| `permission_denied` | 403 OR (RequireRole already rejected — defensive 403 only) | ForbiddenView |
| `success` | 200 + populated | KPI tiles + HairlineTable by model |

`error_validation` is **N/A** — page submits no form; backend 422 cannot happen with the
client's hard-coded `group_by=model` + valid 30-day window. Document as `error_validation: N/A
(read-only, client guarantees valid params)` in the handoff. If 422 is ever observed it falls
through to `error_network` with a generic error message + retry (defensive — log it loud).

### D-T001-401-VS-403

- 401: `authFetch` already attempted refresh; on final 401 → call `onAuthFailure()` (from
  AuthProvider) which clears session and bounces to `/auth/sign-in?next=/admin`.
- 403: render `ForbiddenView` (`admin-ai:dashboard.errors.permissionDenied`). This is defensive
  — in practice RequireRole prevents reaching the fetch for a non-admin.

### D-T001-LOGS-PII-CLEAN

Frontend logs MUST NOT include user email, full name, model API keys, prompt text or
PII. Allowed: user_id (UUID), counts (rows.length, invocations), latency averages, query
timing, error class name.

### D-T001-FILESIZE-SPLIT (proactive)

The page renders 5 states. Anticipate ~250+ LoC in `AdminDashboardPage.tsx` if all states live
inline. Pre-authorize a sibling `_AdminDashboardPage.error-views.tsx` for error/empty/permission
sub-components (T002/T003/T004 precedent). If the page stays ≤220 substantive lines, the split
is unnecessary — judgement call by developer.
