### 2.10 ADRs relevant to this slice

- **ADR-001 (Refresh cookie Path = `/api/v1/auth`)** — `_REFRESH_COOKIE_PATH` constant; refresh works only under `/api/v1/auth/*`. For this slice it means `authFetch` calls to `/api/v1/users/me` and `/api/v1/chat/conversations` do NOT send the refresh cookie (intended; only access-token Bearer). 401 recovery still works because `_doRefresh()` explicitly POSTs `/api/v1/auth/refresh`.
- **ADR-002 (Same-origin reverse proxy in dev via vite + prod via Nginx)** — `VITE_API_BASE_URL=""` is the contract; the client uses paths relative to current origin. `httpClient.ts` currently uses `import.meta.env.VITE_API_BASE_URL ?? "http://localhost:8000"` — the dev default is the absolute URL, which works in dev but the production contract is `""`. For new chat API calls reuse `authFetch` so the strategy is centralized.

### 6.2 POST /api/v1/chat/conversations (P02-S03-T001, deployed)

- Method: `POST` · Path: `/api/v1/chat/conversations` · Auth: `Bearer <access_token>` (employee role).
- Request body (JSON): `{initial_message?: string, language?: "es"|"en"|"fr"}`. Both optional per TECHNICAL_GUIDE §6.2; the backend `app/chat/services/create_conversation.py` enforces D-LANG1 (falls back to user.preferred_language) and D-TIT1 (title derived from `initial_message` if present).
- Response 201: `{data: {conversation_id: UUID, title: string, language: "es"|"en"|"fr", created_at: ISO8601}}` (D-TX1 — atomic write: conversation + first message persisted in one transaction).
- Errors:
  - `400` validation error (e.g. invalid language).
  - `401 AUTH_SESSION_EXPIRED` — handled by `authFetch` single-flight.
  - Any 5xx → surface error_network state.
- Side effects: writes to `conversations` (one row) and `messages` (one row when `initial_message` is present). No streaming.

Concrete frontend usage:

```ts
const body = {
  initial_message: prompt.trim(),
  language: user?.preferred_language ?? "es",
};
const resp = await authFetch("/api/v1/chat/conversations", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(body),
}, { onAuthFailure });
if (!resp.ok) {
  // 400 → validation error state; 5xx/network → error_network state.
}
const data = await resp.json() as { data: { conversation_id: string } };
navigate(`/chat/${data.data.conversation_id}`);
```
