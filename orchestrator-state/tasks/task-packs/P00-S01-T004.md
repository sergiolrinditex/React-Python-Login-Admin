### C.1 Canonical tokens (TECHNICAL_GUIDE line 364)

| Token | Value | Notes |
|---|---|---|
| `--color-bg` | `#f5f3ee` | Crude background (editorial Inditex/Zara codex). |
| `--color-ink` | `#0a0a0a` | Tinta negra para texto y CTAs. NOT pure black `#000`. |
| `--color-paper` | `#ffffff` | Pure white for sheets/cards (but no rounded corners). |
| `--font-display` | `Bodoni Moda, Didot, serif` | Wordmark + assistant copy in chat. |
| `--font-sans` | `Helvetica Neue, Arial, system-ui, sans-serif` | Body + UI + user copy in chat. |
| `--hairline` | `1px solid rgba(10,10,10,.22)` | Single canonical border. NO other border styles. |
| `--tracking-label` | `.18em` | Uppercase tracked labels. |
| `--radius` | `0` | **Zero** rounded corners anywhere. Hard rule. |

### C.2 Prohibitions (UX_CONTRACT §5 + instrucciones §7)

1. **NO `border-radius` > 0** on any element. Forms, inputs, CTAs, cards, modals — all square.
2. **NO `box-shadow`** for decoration. Only neutral hairlines.
3. **NO colored badges** beyond monochrome `StatusDot` (a dot + uppercase tracked label).
4. **NO bubble styling** in chat — assistant copy is serif, user copy is sans-serif, separated by tracked labels, NOT by rounded backgrounds.
5. **NO CTA colors** other than black ink on crude/white background.
6. **NO hardcoded font-family** outside the tokens file.
7. **NO hex colors** in product code outside `frontend/src/shared/styles/`.
8. **NO decorative invented business data** in showcase — use real or provided fixtures; for component-level demos use the explicit demo fixtures listed in §E.

### C.3 Required base components (TECHNICAL_GUIDE §7 line 364 + UX_CONTRACT §5 item 2)

UX_CONTRACT §3 mandates required states per route. `/showcase` is NOT a journey route; it is a design-system demo surface. | Component | States to demo on /showcase | Provided/fixture data |
|---|---|---|
| `Wordmark` | default | none — pure brand mark |
| `TrackedLabel` | default, active (with `StatusDot`), muted | declared literal strings |
| `EditorialInput` | empty, filled, focused, error_validation, disabled | declared literal placeholder/value strings |
| `SolidCTA` | default, hover (CSS only, no JS), pressed, disabled, loading | none required |
| `HairlineTable` | empty, populated (3 rows), error_network (with retry CTA), permission_denied | static rows: `{model, status, latency}` triplets declared inline; flagged as `demo_fixture` |
| `StatusDot` | active, inactive, syncing, error | declared label strings |
| `MobileFrame` | default with placeholder copy | static placeholder copy in English |
| `AdminShell` | default with placeholder left-nav items | static placeholder labels |
| `CitationInline` | default, hover | static label `[Fuente 1]` |

**States explicitly N/A for /showcase (with reason):**

- `loading` page-level — showcase loads synchronously from static fixtures, no async. Component-level loading state (`SolidCTA loading`) IS shown.

- `permission_denied` page-level — showcase is dev-only and not auth-guarded.
- `streaming/uploading/indexing/syncing` page-level — those are journey-specific.

`real_data_or_backend_used: no — design-system showcase uses declared fixtures only (not decorative invented business data, not journey data)` is the line for `VISUAL_CONTRACT_CHECK`.
