### 7.1 Compose-level invariants this script must honor

1. **Entrypoint is `/bin/sh`** (not bash). Stick to POSIX sh.
2. **Inputs come from env only.** Three required: `MINIO_ROOT_USER`, `MINIO_ROOT_PASSWORD`, `S3_BUCKET_DOCUMENTS`. No CLI args, no `.env` reading.
3. **MinIO endpoint inside the docker network**: `http://minio:9000`. Do NOT use `localhost`. Do NOT make this configurable (it would be over-engineering; if the service rename ever happens, edit both compose + script in the same diff).
4. **Idempotent**: running twice in a row must succeed both times (`--ignore-existing` on `mc mb`).
5. **Exit code discipline**: exit 0 on success (bucket exists or was created). Non-zero on real failure (bad creds, mc unreachable, alias error, mb error not caused by "already exists").
6. **No loop**: compose has `restart: "no"`, so any non-zero exit stays as a failed sidecar visible in `docker compose ps`.
7. **No interactive prompts** (no `read`, no `mc --interactive`).

### 7.2 Env vars contract (from `.env.example` lines 33, 46–52)

```
S3_BUCKET_DOCUMENTS=hilo-docs-dev          # default; can be overridden per env
MINIO_ROOT_USER=hilo                       # dev default; rotate in staging/prod
MINIO_ROOT_PASSWORD=hilo-dev-only          # dev default; rotate in staging/prod
```

Compose passes them through the `environment:` block of `minio-init` (lines 143–146 of `docker-compose.yml`). The script must fail fast and clearly if any of the three is empty (avoid silent `mc alias set hilo http://minio:9000 ""` which would create a broken alias).

### 9.5 Unknowns / risks

- **R1** — POSIX vs bash drift: if developer reflexively writes `#!/usr/bin/env bash` with `set -euo pipefail` + `[[ ]]`, compose's `/bin/sh` entrypoint will partially execute then fail on the first bashism. Fix: shebang `#!/bin/sh`, `set -eu`, `[ ]` syntax. (Shebang is informational here because compose explicitly invokes `/bin/sh script`, but pin it for clarity if someone ever runs `./scripts/minio-bootstrap.sh` directly inside a real bash shell.)
- **R2** — Bind-mount path mismatch: compose mounts to `/scripts/minio-bootstrap.sh` (singular, leading slash, no trailing). The script path inside the container is that exact path. Do NOT rename the file; do NOT add subdirectory. Filename literal: `minio-bootstrap.sh` in `scripts/` at repo root.
- **R3** — `mc` version drift in `minio/mc:latest`: the tag is mutable; in CI you may pull a newer `mc` that changes flag names. Researcher Q1+Q2 (below) must confirm `mc alias set` and `mc mb --ignore-existing` still behave as expected on a 2025/2026 `mc` release. If a flag changes, prefer compose-level pin to a digest over patching the script.
- **R4** — Ready-check race: even with `depends_on: service_healthy`, the first `mc` call can hit a transient 503 if MinIO's HTTP server is responding to healthcheck `/minio/health/live` faster than its admin/S3 surface comes up. Researcher Q3 covers the official recommendation (likely `mc ready` or a short retry loop).
- **R5** — `restart: "no"` means a transient failure does not auto-retry. A retry loop INSIDE the script is the right place for resilience.
- **R6** — Secret leakage: if `mc alias set` echoes the secret on error, it ends up in compose logs. Always pass creds via the args, never via env-substitution-into-stdout.
