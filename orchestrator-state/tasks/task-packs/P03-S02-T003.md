## 2. Endpoint contract — GET /api/v1/chat/conversations

### 2.1 Request

```
GET /api/v1/chat/conversations?cursor=<base64>&limit=<n>
Authorization: Bearer <access>   (added by authFetch)
X-Request-ID: <uuid v4>          (added by authFetch)
credentials: 'include'           (refresh cookie)
```

- `cursor` (optional): opaque base64url; if omitted → first page.
- `limit` (optional): backend default; do not hardcode a magic number on the front — pass `undefined` until paging is exercised.
- Same-origin URL only (ADR-002). Relative path; never hardcode `http://localhost:8000`.

### 2.2 Response (200)

```json
{
  "data": [
    {
      "id": "<uuid>",
      "user_id": "<uuid|null>",
      "title": "<str>",
      "language": "es|en|fr",
      "created_at": "<iso8601 utc>",
      "updated_at": "<iso8601 utc>"
    }
  ],
  "meta": {
    "request_id": "<uuid>",
    "pagination": { "next_cursor": "<base64|null>", "has_more": true }
  },
  "errors": []
}
```

### 2.3 Error paths

- `401` → access expired → `authFetch` triggers single-flight refresh; on refresh fail emits `onAuthFailure()` so `RequireAuth` redirects to `/auth/sign-in?next=/history`.
- `403` (impossible for own user list, but handle defensively) → `ChatForbiddenError` → `ForbiddenView`.
- Network failure / 5xx → `ChatNetworkError` → `NetworkErrorView` with retry CTA.

### 2.4 Pagination policy for this slice

Default UI loads first page only. Infinite scroll / "load more" is **out of scope** for T003 (no UX_CONTRACT requirement, no acceptance line for it, and no Coverage Registry "infinite" call-out). If the response says `has_more: true`, surface a discreet "next" affordance or omit entirely — choose omit for v1 to keep the surface tight and add a follow-up only if `/verify-slice` confirms it is needed. See `## 12 Risks` for the decision rationale.

### 4.1 Relative-date grouping

Buckets, computed from `updated_at` in user's local timezone (browser `Intl.RelativeTimeFormat` or hand-rolled with tokens):

1. **Today** (same calendar day)
2. **Yesterday** (calendar day - 1)
3. **This week** (within last 7 days, excluding today/yesterday)
4. **This month** (within last 30 days, excluding above)
5. **Earlier** (everything older) — labeled with month-year format (`MMMM YYYY`)

i18n keys: `history.groups.today`, `history.groups.yesterday`, `history.groups.thisWeek`, `history.groups.thisMonth`, `history.groups.earlier`. Hairline separator between groups (`--hairline` token only).
