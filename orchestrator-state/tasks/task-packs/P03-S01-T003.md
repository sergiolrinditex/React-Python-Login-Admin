## 4. Backend contract ‚Äî exact wire shape (from `backend/app/auth/routers/forgot_reset.py` + `schemas.py` + `errors.py` + `rate_limit.py`)

**Request**: `POST /api/v1/auth/forgot-password` Content-Type: application/json

```json
{ "email": "employee@inditex-sandbox.com" }
```

- Validation: Pydantic `EmailStr` (RFC 5322 syntax only, no domain restriction; corporate-email check happens implicitly via "user must exist" ‚Äî but server returns same 200 for unknown so frontend sees no enumeration signal).
- Rate-limit: 3/min/IP (`AUTH_FORGOT_RATE_PER_MINUTE`, env-overridable).
- Logging: backend logs `ip_prefix` + `request_id` only; never email.

**Response shapes**:

| Status | Body | Headers | Frontend handling |
|---|---|---|---|
| 200 | `{"data":{"sent":true},"meta":{"request_id":"<uuid>"},"errors":[]}` | ‚Äî | Success ‚Äî navigate to `/auth/reset-sent` (T004 placeholder; OK to navigate to `/auth/sign-in` if /auth/reset-sent route does not yet exist ‚Äî see ¬ß6 D-T003-SUCCESS-REDIRECT) |
| 400 | `{"errors":[{"code":"<pydantic>","field":"email","message":"‚Ä¶"}]}` | ‚Äî | Email syntax invalid ‚Äî should be caught client-side by zod first; map to `error_validation` for defence-in-depth |
| 429 | `{"errors":[{"code":"AUTH_FORGOT_RATE_LIMITED","message":"Too many reset requests; retry after 60s"}]}` | `Retry-After: 60` | Rate-limited ‚Äî show `error_validation` with countdown (no `permission_denied` in this slice's UI states) |
| 5xx | `{"errors":[‚Ä¶]}` | ‚Äî | Server error ‚Äî map to `error_network` (best UX bucket since success is the contract) |
| TypeError (network) | ‚Äî | ‚Äî | NetworkError ‚Üí `error_network` |

**Anti-enumeration invariant** (TECHNICAL_GUIDE ¬ß6.2 + backend code line 68 `_FORGOT_SUCCESS_DATA = {"sent": True}`): unknown email, deactivated user, deleted user, mail service down ‚Äî all return the **same 200 body**. Frontend MUST NOT try to differentiate. Show identical success state for all 200 responses.
