### Backend response shape (autoritativo desde `backend/app/chat/schemas.py` línea 220 y siguientes)

```typescript
// GET /api/v1/chat/conversations/{id} → 200
{
  data: {
    id: string;            // UUID
    user_id: string | null;
    title: string;
    language: 'es' | 'en' | 'fr';
    created_at: string;    // ISO-8601 UTC
    updated_at: string;
    messages: Message[];   // ordered by created_at asc
    citations: MessageCitation[];
  },
  meta: { request_id: string },
  errors: []
}

interface Message {
  id: string;
  conversation_id: string;
  role: 'user' | 'assistant' | 'system';
  content: string;
  token_count: number | null;
  created_at: string;
}

interface MessageCitation {
  id: string;
  message_id: string;
  document_id: string | null;
  chunk_id: string | null;
  label: string;
  score: number;
}
```

### SSE wire format (`POST .../stream`)

W3C `text/event-stream`. Order en V1 (backend `app/chat/streaming/service.py::run_stream`):

```
event: meta
data: {"message_id":"...","model_id":"...","language":"es","request_id":"..."}

event: citation                        (0..N, antes de chunks)
data: {"document_id":"...","chunk_id":"...","label":"...","score":0.87}

event: chunk                           (1..N)
data: {"delta":"..."}

event: usage
data: {"prompt_tokens":...,"completion_tokens":...,"total_tokens":...,"cost_usd":...}

event: done
data: {"message_id":"..."}

# o, en lugar de done:
event: error
data: {"code":"STREAM_ERROR","message":"..."}
```

**§D-T002-STREAM-DONE-GUARD** (defect del referencial f7f5f33): si el stream se cierra prematuramente SIN `event: done` ni `event: error`, el parser debe transicionar a `error_network` (no a `completed`). Sin este guard el referencial inicial dejaba la página atascada en `streaming` para siempre. Replicar el guard en la re-implementación.
