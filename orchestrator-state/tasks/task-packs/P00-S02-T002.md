### request_id correlation middleware (in scope, minimal)
- Read header `X-Request-ID`; if missing, generate `uuid.uuid4().hex`.
- Bind into structlog contextvars: `structlog.contextvars.bind_contextvars(request_id=<value>)`.
- Echo back as `X-Request-ID` on the response.
- Clear contextvars at end of request.
- Live in `backend/app/api/router.py` would mix concerns — **belongs in a small helper module**. The work-item write_set lists 3 files; adding a new module would extend write_set. Recommended path: declare the middleware as a function inside `app.api.router` (cohesive: same ops surface) OR keep it inline in `main.py` (4–6 lines). **Preferred: inline middleware in `main.py`** (same write set, smaller surface). Document in the developer's handoff.

### Risks
- **R1 — Inline /health refactor regression**. Test the flat-shape contract precisely so the existing curl `{"status":"ok","version":"0.1.0","uptime":<n>}` still parses. Keep the JSON keys, types and order semantically identical (order isn't guaranteed in JSON dicts but the keys must be exactly those three).
- **R2 — DB probe false positive**. `pool_pre_ping=True` is already set in `core/db.py`; a `SELECT 1` after `pool_pre_ping` is sometimes redundant. Run the explicit `SELECT 1` anyway because pool_pre_ping only fires on checkout. Document in handoff.
- **R3 — Engine creation at startup**. `get_engine()` is lazy — it builds the engine on first call. `/ready` will be the first caller in production. Acceptable; ensure the test for DB-down does NOT leak a half-built singleton into other tests (use `monkeypatch` on the module-level `_engine` global, or refactor handler to accept a `engine_factory` callable for DI).
- **R4 — request_id middleware exception path**. If structlog contextvars binding fails, must not 500 the request. Wrap in try/except that logs at WARNING and continues without the bound context.
- **R5 — Port shadow from T003 dev venv**. T001 closer documented a leftover `.venv-t003` uvicorn on `127.0.0.1:8000` shadowing the container. This is environmental; tests use in-process `httpx.AsyncClient`, so they are immune. The verification curl in the acceptance command may hit either; document in handoff which process responded.
