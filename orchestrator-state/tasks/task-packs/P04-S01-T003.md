### 6.3 Risks

1. **Secret leakage** — accidental logging or rendering of `secret_plain`. Mitigation: keep secret_plain in a `useRef` or `useState` whose value is never serialized to console; mask in UI as `••••• last4`; explicitly verify with test `expect(consoleSpy).not.toHaveBeenCalledWith(expect.stringMatching(/hilo-dev-litellm/))`.
2. **Persistence beyond submit** — DO NOT write `secret_plain` to `localStorage` or `sessionStorage` (non-negotiables §security/Web): keep it only in component state. After submit (success or failure), clear the field by resetting state.
3. **Router wiring drift** — page exists but no `<Route>` (current state). Mitigation in file #6 above.
4. **i18n drift** — adding ES keys but forgetting EN/FR fails the lockstep test in `frontend/src/i18n/__tests__/i18n.test.ts`. Mitigation: add all three locales in the same commit.
5. **422 field-error parsing** — backend returns `{error, code, details}` envelope; the wizard must surface field-level errors next to the offending input (per non-negotiables §error-handling). Mitigation: parse `body.errors[]` into a `Record<field, message>` and pass to `EditorialInput` `error` prop.
6. **Models step empty after create** — verification provider seed has no models pre-registered; `GET /api/v1/admin/ai/models?provider_id=<new>` likely returns `[]`. The wizard must render an explicit empty state with copy "No models discovered yet" and a CTA back to `/admin/ai/models` (next action). This is acceptance criterion 1 ("lists models … validates fields"). Mitigation: pre-create model row in verification seed via loader if `--only admin_ai` includes it, otherwise document in handoff and leave the empty state visible during verify.
7. **TanStack Query cache staleness** — after POST success, the list at `/admin/ai/models` must show the new provider. Mitigation: in `useModelWizard.submit.onSuccess`, call `queryClient.invalidateQueries({queryKey: ['admin','ai','models']})` so AdminAiModelsPage refetches when the user navigates back.
8. **D-T003-PROVIDER-OPTIONS i18n keys missing test catch** — extending i18n test suite is out of write_set but locking shapes is needed; mitigation: add a `modelsNew` lockstep test in the same `i18n.test.ts` (T002 already added `models.*`; keep the pattern).
9. **Audit log creation** — backend `service.create_provider` already writes audit row; the wizard does NOT need to create it. Mitigation: verify the row appears in DB during `/verify-slice`.
10. **Provider name uniqueness** — backend may return 409 on duplicate name (defensive). If observed, surface as `error_validation` with field=`name`, message from server.
