## Riesgos y mitigaciones

1. **Risk**: developer expands scope by installing real deps "to make verify pass". **Mitigation**: `verification_commands=[bash scripts/setup-from-scratch.sh --check]` only checks declared commands resolve; `--check` mode tolerates `frontend.npm install` not yet wired (deps land in T002). If `--check` fails because of missing deps, that's a real signal — surface as `OUTCOME: blocked` and pass to T002, do NOT install here.
2. **Risk**: developer creates auth/chat/admin scaffolding "as placeholders". **Mitigation**: write-scope-guard hook + the Allowed Paths list above. Out-of-scope table makes ownership explicit.
3. **Risk**: `.env.example` leaks real secrets. **Mitigation**: every value is a placeholder (`<change-me>`, `localhost`, ports). Real secrets land in `.env` (untracked) per `01-non-negotiables.md#security`; `01-non-negotiables.md` forbids secrets in source.
4. **Risk**: `backend/app/main.py` is "valid" but breaks future slices' router pattern. **Mitigation**: keep `main.py` ≤ 30 lines, just `app = FastAPI(title="Hilo People", version="0.0.0")` + a single `/health` route. T003 introduces `app/core/config.py`; P00-S02-T002 splits the route into `api/router.py`. Both T003 and P00-S02-T002 own those refactors, not this slice.
