## 7. Logging contract (from `01-non-negotiables.md` §Logging)

The `/health` stub must log BEFORE and AFTER:

- BEFORE: `health.check.start` — what is about to be done (e.g. `{"event":"health.check.start","route":"/health"}`).
- AFTER: `health.check.ok` with the returned status (e.g. `{"event":"health.check.ok","status":"ok"}`).
- On error (unexpected): `health.check.error` with stack trace context — never log secrets/PII.

`ENABLE_VERBOSE_LOGGING` semantics (from non-negotiables, mirrored in `.env.example`):

- `true` → BEFORE + AFTER + errors visible.
- `false` → warnings and errors only; BEFORE/AFTER suppressed.
- The CODE never changes between modes — only the flag. Do NOT add `if ENABLE_VERBOSE_LOGGING:`
  guards around log statements; use a logger level filter driven by the env var.

For this slice a minimal stdlib `logging` configuration is sufficient (`structlog` lands in
T003 alongside the backend dep pack). Keep it simple: `logging.getLogger(__name__).info(...)`.
