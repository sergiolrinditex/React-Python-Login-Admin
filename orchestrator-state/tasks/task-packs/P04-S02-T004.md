### 3.3 Backend POST contract (from `backend/app/mcp/router_servers.py` lines 104–181 + `backend/app/mcp/schemas.py` lines 64–86)

```http
POST /api/v1/admin/ai/mcp/servers
Authorization: Bearer <access_token>   (single-flight refresh handled by authFetch)
X-Request-ID: <UUID v4>                (auto-injected by authFetch)
Content-Type: application/json
```

Request body (`CreateServerRequest`, `extra="forbid"`):
```json
{
  "name": "string (1..200)",
  "transport": "http" | "sse",          // stdio is REJECTED by Pydantic (422); UI MUST hide stdio
  "endpoint": "string (1..2000)",       // URL of MCP HTTP/SSE endpoint
  "auth": {
    "type": "none" | "api_key" | "bearer" | "oauth2",
    "secret": "string | null",          // required (non-empty) unless type='none'
    "refresh_token": "string | null"    // optional, only meaningful for oauth2
  }
}
```

Responses:
- `201 Created` → `{ data: ServerOut, meta: { request_id } }` — see §3.4 below for ServerOut.
- `400 Bad Request` → `{ data: null, meta: { request_id }, errors: [{ code: "MCP_ENDPOINT_NOT_ALLOWED", message, field: null, details: null }] }` — endpoint host not in `MCP_ALLOWLIST_DOMAINS` env var (in dev with empty allowlist this is bypassed, but real envs reject). UI MUST surface inline as `error_validation` near endpoint field.
- `401 Unauthorized` → standard authFetch single-flight refresh handles; if exhausted → `McpAuthExpiredError` → call `onAuthFailure` (logout/redirect).
- `403 Forbidden` → in-page permission_denied (defensive; route guard catches this already).
- `422 Unprocessable Entity` → FastAPI/Pydantic native shape:
  ```json
  { "detail": [{ "loc": ["body","auth","secret"], "msg": "Field required", "type": "missing" }, ...] }
  ```
  UI MUST map each `loc` to RHF `setError(<fieldName>, ...)` so the field highlights inline. **Client-side Zod schema MUST mirror Pydantic so 99% of 422s never happen — they remain a backend safety net.**
- `429 Too Many Requests` / `503 Service Unavailable` → rate limiter (`register_server_limiter`). UI shows inline form-level error "Demasiados intentos, espera unos segundos". This branch IS reachable in real runs — `slice-verifier` will not stress it but the page MUST not blank.
- `500 Internal Server Error` → encryption failure (Fernet) or unexpected exception. Map to `McpServerError(500)` → inline form-level error.

Server response on success (`ServerOut`, line 92 of `schemas.py`):
```json
{
  "id": "<UUID>",
  "name": "string",
  "transport": "http" | "sse",
  "endpoint": "string | null",
  "status": "draft",                    // newly registered servers start in 'draft' status
  "last_sync_at": null,
  "created_by": "<UUID>",
  "has_credential": true | false,       // true unless auth.type='none'
  "auth_type": "api_key" | "bearer" | "oauth2" | null   // null when type was 'none'
}
```

### 3.4 DB schema (TECHNICAL_GUIDE §10.3, lines 481–485)

```sql
CREATE TABLE mcp_servers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  transport_type TEXT NOT NULL,         -- 'http' | 'sse' for external servers
  endpoint_url TEXT,                    -- nullable in DB; required in request for http/sse
  command TEXT,                         -- stdio only; UNUSED for external
  status TEXT NOT NULL DEFAULT 'draft',
  last_sync_at TIMESTAMPTZ,
  created_by UUID REFERENCES users(id)
);
CREATE TABLE mcp_credentials (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  server_id UUID REFERENCES mcp_servers(id) ON DELETE CASCADE,
  auth_type TEXT NOT NULL,
  encrypted_secret TEXT,                -- Fernet (cryptography.Fernet)
  encrypted_refresh_token TEXT,         -- Fernet
  expires_at TIMESTAMPTZ
);
```

Wizard is READ-ONLY from the DB's perspective: only the backend writes rows. Frontend MUST never touch DB directly. After successful 201, **frontend MUST not display the secret again, MUST not store it in any client state beyond the form lifetime, and MUST not log it (even at length).**

### 3.5 Cross-cutting non-negotiables this slice MUST honor (`.claude/rules/01-non-negotiables.md`)

- **Token storage**: Web context. Refresh in HttpOnly cookie (already wired via `authFetch`). Access token in memory only. **The MCP server credential the admin types into the wizard is NOT an auth token of the user — it's a third-party provider key.** Per non-negotiables §"Claves de proveedores externos":
  - NEVER log the secret (not even its length unless absolutely necessary). The mcp logger MAY log `auth_type` value and `field name` only.
  - NEVER persist in localStorage / sessionStorage / IndexedDB.
  - Backend handles Fernet encryption; admin panel never receives it back masked (POST response has `has_credential: bool` only — never the secret).
  - Inputs of type `password` (with `autoComplete="off"` + `spellCheck={false}`) — never `type="text"` for the secret.
- **Audit log**: Backend already writes `mcp_server_registered` to `audit_logs` (TECHNICAL_GUIDE §"Audit log"). Frontend has NO duty here beyond sending the X-Request-ID through `authFetch`.
- **Request correlation**: `authFetch` injects `X-Request-ID`. Read it back from response headers and include in error logs (`request_id` field) — already the pattern in `mcpRepository.ts:84`.
- **Logging**: BEFORE/AFTER/ERROR for `createServer`, `useCreateMcpServer.mutate`, and page submit. Verbose-gated by `import.meta.env.VITE_ENABLE_VERBOSE_LOGGING === "true"`. Logs MUST NOT contain: secret, refresh_token, endpoint URL with credentials (full URL is allowed only if it does NOT carry credentials in the userinfo segment; safest is to log the host portion only — see how `mcpRepository.ts` already logs only IDs, counts and request_ids).
- **A11y**: every field has linked `<label>` via `EditorialInput.label` prop. Error messages linked via `aria-describedby` (the `errorMessage` prop wires this). Min tap target ≥44px. Keyboard-only flow: TAB through name → transport → endpoint → auth type → secret/refresh → Submit. Submit must work with Enter key on the last field.

## 8. Wire contract (CreateServerRequest shape — frontend Zod schema)

```ts
import { z } from "zod";

// Mirrors backend Pydantic CreateServerRequest exactly (schemas.py:64).
// Wizard MUST NOT allow stdio (instrucciones §3.1 line 99; backend rejects with 422).

export const McpTransportEnum = z.enum(["http", "sse"]);   // NOT including "stdio"
export const McpAuthTypeEnum = z.enum(["none", "api_key", "bearer", "oauth2"]);

export const CreateServerFormSchema = z
  .object({
    name: z.string().min(1, "wizard.errors.nameRequired").max(200, "wizard.errors.nameMax"),
    transport: McpTransportEnum,
    endpoint: z
      .string()
      .min(1, "wizard.errors.endpointRequired")
      .max(2000)
      .refine(
        (v) => /^https?:\/\//i.test(v) || /^sse:\/\//i.test(v),
        "wizard.errors.endpointInvalid"  // soft check — backend allowlist is authoritative
      ),
    authType: McpAuthTypeEnum,
    secret: z.string().optional(),
    refreshToken: z.string().optional(),
  })
  .superRefine((val, ctx) => {
    if (val.authType !== "none" && !val.secret) {
      ctx.addIssue({
        code: z.ZodIssueCode.custom,
        path: ["secret"],
        message: "wizard.errors.secretRequired",
      });
    }
  });
```

`CreateServerRequest` (the wire DTO, passed to repository) MUST shape the payload as:
```ts
const wireBody: CreateServerRequest = {
  name: formValues.name,
  transport: formValues.transport,
  endpoint: formValues.endpoint,
  auth: {
    type: formValues.authType,
    secret: formValues.authType === "none" ? null : formValues.secret ?? null,
    refresh_token: formValues.authType === "oauth2" ? formValues.refreshToken ?? null : null,
  },
};
```
