## 8. Impact analysis

### Root causes (from FU-20260509130036)

#### Root cause #1 — DATABASE_URL host port mismatch in `.env.example`

**Current state** (`.env.example` L35):

```dotenv
DATABASE_URL=postgresql+asyncpg://hilopeople:<change-me>@localhost:5432/hilopeople_dev
```

**Reality**:
- `hilo-postgres` (the compose service this project runs) maps host **5433** → container 5432
  (docker-compose.yml L72: `"5433:5432"`).
- Port 5432 on the host belongs to a sibling project's `apn-postgres`. Connecting there
  with this project's credentials returns **InvalidPasswordError** — exactly the symptom
  PROGRESS.md "1 pre-existing auth fail" describes.

**Two valid usage modes** (must both be documented):

| Mode | Caller | Host | Port | Password |
|---|---|---|---|---|
| A. Native dev (uvicorn from host, alembic from `cd backend/`, pytest from `cd backend/`) | host process | `localhost` (or `127.0.0.1`) | **5433** | `hilopeople_dev_pwd` |
| B. Inside compose (backend container, worker container) | container | `postgres` (service name) | **5432** | `${POSTGRES_PASSWORD}` |

`.env.example` must document BOTH and ship Mode A as the default uncommented value
(because the ONLY use case that reads `.env.example` directly is a developer
copy-pasting it to start native dev — Mode B is set inline in `docker-compose.yml`
and does not depend on `.env`).

#### Root cause #2 — pydantic-settings `env_file=".env"` is cwd-relative

**Current state** (`config.py` L62-67):

```python
model_config = SettingsConfigDict(
    env_file=".env",
    env_file_encoding="utf-8",
    case_sensitive=False,
    extra="ignore",
)
```

**Behavior** (verified against pydantic-settings 2.14.1 docs):
- `env_file` is interpreted **relative to the process cwd at Settings() instantiation time**.
- ✅ Works when uvicorn runs from project root (`cwd=/.../React-Python-Login-Admin`, `.env` is there).
- ❌ Breaks when alembic/pytest run from `cd backend/` (`cwd=/.../React-Python-Login-Admin/backend`,
  no `.env` there → silently falls back to defaults: `database_url` default points to
  `localhost:5432` with password `change-me` → InvalidPasswordError).

**The fix** (idiomatic pydantic-settings 2.14.1):

Resolve the project root deterministically from `__file__` and pass an absolute `Path`:

```python
from pathlib import Path

# config.py is at: backend/app/core/config.py
# parents: [0]=core, [1]=app, [2]=backend, [3]=<project_root>
_PROJECT_ROOT = Path(__file__).resolve().parents[3]
_ENV_FILE = _PROJECT_ROOT / ".env"

class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_file=str(_ENV_FILE),  # absolute — cwd-independent
        env_file_encoding="utf-8",
        case_sensitive=False,
        extra="ignore",
    )
```

Notes for the developer:
- `parents[3]` is the canonical anchor for `backend/app/core/config.py`. Verify with a comment.
- pydantic-settings accepts `Path | str | os.PathLike | list[...]` for `env_file`. A single
  `str` is the simplest. A `list[Path]` would also work and would let us layer
  `~/.config/hilopeople/.env` later if needed — out of scope for this slice.
- Existing process-env vars (real environment variables set by docker-compose, CI, etc.)
  STILL win over `.env` file values. This preserves Mode B behavior in compose.
