## Reglas de negocio y UX (instrucciones.md + UX_CONTRACT)

Invariantes para esta slice (instrucciones.md §3.1#admin-ai, líneas 71–79):

2. **Solo admins activan modelos**. `require_admin` en los 4 endpoints (cubre `people_admin` + `super_admin`).
3. **Default models invariant** (instrucciones §3.1#admin-ai): "debe existir un modelo chat default y un modelo embeddings default". Esto se traduce en: **at-most-one `is_default=true` por `model_type` (chat/embeddings) en `ai_models`**. Hoy NO existe partial unique index en la DB (ver §Riesgos R-IDX abajo). Enforcement provisional a nivel de aplicación dentro de PATCH (transacción que ponga otros a `is_default=false` antes de poner el nuevo a `true`). Documentar en handoff como **D-DEF1: app-level invariant pending DB-level partial unique index** (FU si procede).
4. **Ninguna API key se devuelve al frontend**. Schemas response NO deben tener `encrypted_secret`, `secret_plain` ni `credentials.*`. El response masked-display (`sk-****1234`) lo hace el frontend; esta slice api-only se limita a NO devolver el campo. Si el wizard lo necesita en P04-S01-T003, devolver `{"masked": "sk-****1234"}` opcional — definir en pack P02-S05-T001 sólo si simplifica P04; por defecto, NO incluir.
5. **Audit log obligatorio** en operaciones mutadoras: POST `/providers` y PATCH `/models/{id}`. Acciones canónicas propuestas (validar con tests):
   - `admin.ai.provider.create` (actor_user_id=admin, entity_type=`ai_provider`, entity_id=provider.id, extra_metadata={request_id, ip, user_agent, name, provider_type, outcome})
   - `admin.ai.model.update` (entity_type=`ai_model`, entity_id=model.id, extra_metadata={request_id, ip, user_agent, from:{enabled,is_default}, to:{enabled,is_default}, outcome})
   - **NO** loggear `encrypted_secret`, `base_url` si tiene token, ni nada del payload `credentials`. Sólo metadata semántica.
   - Patrón D-S2: usar `app.db.session.audit_session_scope()` (mirroring `users/audit.py`).
6. **RBAC** admin obligatorio en los 4 endpoints — error `AUTH_PERMISSION_DENIED` (403) byte-equivalente al de auth.
7. **Rate-limit** en POST `/providers` (única operación de creación) — bucket prefix `ADMIN_AI`, `per_minute=20`, `burst=20`, `window_seconds=60`. GET/PATCH no llevan rate-limit per-endpoint (cubre auth global).

### 2) `POST /api/v1/admin/ai/providers`

- Auth: `require_admin`
- Rate limit: `ADMIN_AI` prefix, 20/min/IP, burst 20.
- Request body (Pydantic):
  ```python
  class CreateProviderRequest(BaseModel):
      provider_type: Literal['openai','anthropic','azure','litellm','ollama','google','custom']
      name: str  # min 1, max 200
      base_url: str | None = None  # validate as URL when present
      credentials: ProviderCredentialsInput  # never echoed back

  class ProviderCredentialsInput(BaseModel):
      auth_type: Literal['api_key','oauth2','bearer']
      secret_plain: str  # min 1; NEVER logged; encrypted before persistence
      refresh_token_plain: str | None = None  # only for oauth2
      expires_at: datetime | None = None
  ```
- Response 201:
  ```json
  {
    "data": {
      "id": "uuid",
      "name": "...",
      "provider_type": "...",
      "base_url": "...|null",
      "status": "draft",
      "created_by": "<admin uuid>",
      "has_credentials": true,
      "credential_auth_type": "api_key",
      "expires_at": "...|null"
    },
    "meta": {"request_id": "..."}
  }
  ```
- Errors: 400 `AI_PROVIDER_PAYLOAD_INVALID`, 401, 403, 422 (Pydantic), 429 `RATE_LIMITED`, 503.
- Tx (single SQLAlchemy session):
  1. INSERT `ai_providers` (status default `draft`)
  2. INSERT `ai_provider_credentials` (`encrypted_secret = encrypt_secret(secret_plain)`, optional `encrypted_refresh_token = encrypt_secret(refresh_token_plain)`)
  3. **D-S2** audit (independent session): action `admin.ai.provider.create`, actor=admin.id, entity=(`ai_provider`, provider.id), metadata={request_id, ip, user_agent, name, provider_type, outcome:`success`|`failure`}
- Logging BEFORE/AFTER: log payload schema con redacción (`***REDACTED***` para `credentials.*`).
