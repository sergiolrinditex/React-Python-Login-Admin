### Security non-negotiables for this slice (`.claude/rules/01-non-negotiables.md`)

- **Token storage (web)**: access token lives in MEMORY ONLY via `setAccessToken()` from `frontend/src/features/auth/data/accessTokenStore.ts`. NEVER `localStorage`/`sessionStorage`. Refresh token is `HttpOnly` cookie set by backend — frontend NEVER reads it.
- **PII logging**: NEVER log raw `code`, `challengeToken`, full `email`, password. ONLY log `code_len`, `challenge_token_len`, `email_domain`, `user_id` (UUID), `request_id`, `expires_in`. Pattern is identical to `useSignIn` / `authRepository.signIn` (already enforced by `data/logger.ts`).
- **Anti-enumeration**: backend returns **byte-identical** 401 body for "wrong code", "invalid challenge", "missing secret", "replay" (4 internal failure modes → 1 wire 401 with `code: AUTH_MFA_CODE_INVALID`). UI MUST NOT differentiate. ONE single 401 copy (§7).
- **Request ID**: send `X-Request-ID: crypto.randomUUID()` on every request; mirrors `authRepository.signIn`.
- **HTTPS in production**: irrelevant at slice level (dev uses Vite proxy ADR-002 — `API_BASE = ""`).

## 6. Front → Back → DB contract

### 6.1 — Request shape (DECISIVE — confirmed from `backend/app/auth/schemas.py:347-397`)

```http
POST /api/v1/auth/2fa/verify
Content-Type: application/json
X-Request-ID: <UUIDv4>

{
  "challenge_id": "<the full mfa_challenge_token JWT string>",
  "code": "123456"
}
```

⚠ **Critical contract subtlety**: the request field is named `challenge_id` but the **VALUE is the full JWT** mfa_challenge_token from the sign-in MFA branch. Backend `MfaVerifyRequest` schema (schemas.py:363-372) explicitly documents this naming. Despite the misleading name, do NOT slice it or attempt to extract a DB id. Send it verbatim. **Pin §D-T005-CHALLENGE-FIELD-NAME**.

- `challenge_id`: `str, min_length=30, max_length=2048` (JWTs are ~180 chars; min=30 is a payload guard).
- `code`: `str, min_length=6, max_length=6, must be all digits` (Pydantic `field_validator` rejects non-numeric).

### 6.2 — Response shape on success (200)

```json
{
  "data": {
    "access_token": "<JWT HS256 ~200 chars>",
    "token_type": "Bearer",
    "expires_in": 1800,
    "user": {
      "id": "<UUID>",
      "email": "employee.verification@inditex-sandbox.com",
      "preferred_language": "es",
      "roles": ["employee"]
    }
  },
  "meta": { "request_id": "<UUID>" },
  "errors": []
}
```

Plus `Set-Cookie: <refresh cookie>` with `Path=/api/v1/auth` (byte-identical to sign-in cookie — P01-S02-T011 contract preserved in `_set_refresh_cookie`).

⚠ **`user` here is a SHORTER `MfaUserDto`** (id, email, preferred_language, roles) — NOT the full `UserProfile` (which has `full_name`, `status`, `employee_profile`, `created_at`, `updated_at`). This mirrors P-28 lesson for SignInPage: TwoFactorPage MUST call `fetchMe(accessToken)` after a successful verify to get the full `UserProfile` before invoking `signInAccepted(token, fullUser)`. **Pin §D-T005-USERFETCH-AFTER-MFA**.

### 6.3 — Error responses

Backend re-maps Pydantic 422 → 400 for `/auth/2fa/verify` (`app/main.py:124`).

| HTTP | Envelope code | Internal cause | UI state | UI copy |
|---|---|---|---|---|
| 400 | `AUTH_INVALID_PAYLOAD` | code not 6 digits / challenge < 30 chars | error_validation | "Introduce un código de 6 dígitos." |
| 401 | `AUTH_MFA_CODE_INVALID` | wrong code / invalid challenge / missing secret / replay (AGGREGATE — anti-enum) | error_validation | "Código incorrecto. Vuelve a intentarlo." |
| 410 | `AUTH_MFA_CHALLENGE_EXPIRED` | signature valid but exp < now | permission_denied | "Tu desafío ha expirado. Inicia sesión de nuevo." + auto-redirect to `/auth/sign-in` |
| 429 | `AUTH_MFA_VERIFY_RATE_LIMITED` | > 20/min/IP (burst 5) | permission_denied | "Demasiados intentos. Espera {{seconds}} segundos." (read `Retry-After` header) |
| 5xx | `AUTH_MFA_VERIFY_INTERNAL_ERROR` | unexpected | error_network | "Error interno del servidor. Inténtalo de nuevo más tarde." |
| TypeError | `NETWORK_ERROR` | fetch rejected | error_network | "Sin conexión. Comprueba tu red e inténtalo de nuevo." |
