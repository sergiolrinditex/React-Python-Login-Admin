### From TECHNICAL_GUIDE §11.2 — Build targets

- Frontend: `npm --prefix frontend run build` produces `frontend/dist/` → copied into nginx image.
- Backend: `docker build -f backend/Dockerfile .` (build context = `backend/`) → multi-stage Python image.
- Worker shares backend image; only `command:` differs (`celery -A app.workers.celery_app worker -Q default,documents,embeddings -l INFO`). Worker module path is `backend/app/workers/` per techguide §4 + checklist P02-S04-T002.

### From TECHNICAL_GUIDE §11.3 — Rollback

- Migrations reversible (Alembic up/down).
- Activación de modelo y MCP son toggles; rollback por `enabled=false`.
- Documentos RAG no se borran físicamente.
- **Compose-level rollback**: tagged images (`IMAGE_TAG` build arg) so `docker compose pull && docker compose up -d` can roll back to the previous tag. Volumes never deleted on `down`; require `down -v` (operator decision).

### From TECHNICAL_GUIDE §10.4 — AI stack

- LiteLLM image is the AI gateway. Used by backend (chat, model test) and worker (embeddings).
- pgvector + pgcrypto extensions required (not optional). Compose Postgres image MUST be `pgvector/pgvector:pg18` or equivalent that ships pgvector pre-built; pgcrypto ships with stock postgres so no extra image needed for it.

## Impact analysis

### Modules to write

- `docker-compose.yml` (root) — new file, ~150-220 lines (one file, one responsibility: declare topology). If it exceeds ~300 lines, split into `docker-compose.yml` + `docker-compose.override.yml` for dev profile.
- `backend/Dockerfile` — new file, ~50-90 lines. Multi-stage:
  - Stage `builder`: `python:3.13-slim`, install build deps, `pip install --user -e .` to wheel cache.
  - Stage `runtime`: `python:3.13-slim`, copy site-packages from builder, `useradd hilo`, `USER hilo`, `EXPOSE 8000`, `HEALTHCHECK CMD curl -fsS http://localhost:8000/health || exit 1`, `CMD ["uvicorn","app.main:app","--host","0.0.0.0","--port","8000"]`.
- `frontend/Dockerfile` — new file, ~40-70 lines. Multi-stage:
  - Stage `node-builder`: `node:22-alpine`, `npm ci`, `npm run build` → `dist/`.
  - Stage `runtime`: `nginxinc/nginx-unprivileged:1.27-alpine`, copy `dist/` to `/usr/share/nginx/html/`, copy `infra/nginx/default.conf` for SPA fallback + `/api/v1` reverse proxy, `EXPOSE 8080`, `USER 101` (nginx-unprivileged default), no healthcheck baked in (compose-level only).
- `.dockerignore` files (root, `backend/`, `frontend/`) — exclude `node_modules`, `.venv*`, `.git`, `.env*`, `__pycache__`, `*.pyc`, `dist`, `build`, `.pytest_cache`, `.ruff_cache`, `.mypy_cache`, `orchestrator-state/`, `docs/`, `scripts/`. **Aggressive ignore** is non-negotiable for image size and rebuild speed.
- `infra/litellm/config.yaml` — minimal LiteLLM proxy config (model_list empty for dev, master key from env). LiteLLM image expects this mounted at `/app/config.yaml`.
- `infra/nginx/default.conf` — SPA + reverse proxy. Listens on 8080, root `/usr/share/nginx/html`, `try_files $uri /index.html`, `location /api/v1/ { proxy_pass http://backend:8000; }`.
- `.env.example` — append `MINIO_ROOT_USER`, `MINIO_ROOT_PASSWORD`, `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `AWS_ENDPOINT_URL=http://minio:9000`, `IMAGE_TAG=dev`, `BUILD_VERSION=0.1.0`. ⚠️ This is a write OUTSIDE the declared `write_set` for the slice. **Decision**: T001 keeps strictly to its declared write_set (`docker-compose.yml`, `backend/Dockerfile`, `frontend/Dockerfile`); developer notes the missing env vars in handoff for follow-up via `register-followup` if hooks block the `.env.example` edit. ## Logging requirements

- All containers MUST log to stdout/stderr only. NEVER write logs inside the container filesystem (Kubernetes 12-factor).
- Backend + worker structlog (already wired in T003) emits JSON to stdout; compose default driver `json-file` captures it. `docker compose logs backend` should show structured JSON entries.
- LiteLLM, MinIO, Redis, Postgres use their own logging defaults (text + level prefixes); fine for dev. In Rancher these go to Loki/Datadog/CloudWatch via the cluster log shipper.
- `ENABLE_VERBOSE_LOGGING` env var is consumed by backend; flip to `true` in dev for full flow visibility.

## Design constraints — Rancher / Kubernetes portability (HARD)

User explicitly declared deploy target = Rancher (Kubernetes). Compose stack must be drop-in-portable to Rancher (Helm in P06). These are non-negotiable design constraints encoded by the planner per the user's directive:

1. **Multi-stage Dockerfiles**, runtime stage without toolchains. Builder stage is throwaway; final image is slim.
2. **PID 1 logs to stdout/stderr only** — never to files inside the container. Backend uses uvicorn → structlog → stdout. Worker → Celery → stdout. Frontend → nginx → stdout (configure via `access_log /dev/stdout; error_log /dev/stderr;` in nginx config).
3. **USER non-root** in every image (backend, worker, frontend nginx-unprivileged).
4. **HTTP healthchecks** where possible — they map 1:1 to Kubernetes `livenessProbe` (HTTP GET) and `readinessProbe` (HTTP GET). Currently:
   - backend: HTTP `/health` (will move to `/ready` after T002).
   - frontend nginx: HTTP `/` returning 200.
   - LiteLLM: HTTP `/health/liveliness`.
   - MinIO: HTTP `/minio/health/live`.
   - postgres + redis: only TCP/cmd healthchecks available natively (acceptable; in K8s use TCP probes or sidecar exporters).
5. **No hardcoded host/port inside images.** All config via env vars listed in techguide §11.1; sane defaults only. Backend/worker accept `DATABASE_URL`, `REDIS_URL`, `LITELLM_BASE_URL` etc. — never literal `localhost` or `127.0.0.1` in image config.
6. **Named volumes** for data dirs (`postgres-data`, `minio-data`, optional `redis-data`) — never bind-mounts (`./data:/var/lib/...`) for stateful services. Named volumes map cleanly to Kubernetes PersistentVolumeClaims via Helm. Bind-mounts are dev-only (e.g., readonly config files like `infra/litellm/config.yaml`).
7. **`.dockerignore` aggressive** in each context (root, backend, frontend) — excludes node_modules, .venv*, .git, .env*, __pycache__, dist, build, orchestrator-state, docs, scripts, *.log.
8. **Build args for `IMAGE_TAG` / `BUILD_VERSION`** so CI can tag images consistently (`hilo-people-backend:${BUILD_VERSION}` or pin by digest). Compose `build.args` block plumbs these through.
9. **Restart policy `unless-stopped`** and reasonable **`stop_grace_period: 30s`** for graceful shutdown under `kubectl rollout restart` / SIGTERM. Backend uvicorn handles SIGTERM natively; worker Celery (when wired in P02-S04-T002) needs `--grace-period` config.
10. **No `host.docker.internal`** anywhere — not portable to Linux/Rancher. Use compose service hostnames end-to-end.
11. **No `network_mode: host`** — breaks K8s pod networking model. Default user-defined bridge only.
12. **No privileged mode**, no `--cap-add` unless documented; should be empty for all 7 services.
13. **Image tag immutability**: pin by digest where possible (`image: foo:tag@sha256:...`), or by exact version. Never `:latest`, never `:main` (LiteLLM `main-stable` is acceptable as an upstream tag but pin the digest).
