  4. **Toda invocación queda auditada** — para este slice, "invocación" = registro/sync/patch (no es la invocación runtime de P02-S08).
- §3 (línea 41 acceptance global): "Un admin puede registrar un servidor MCP externo, hacer discovery de tools, aprobar tools y asignarlas a agentes" — esta slice cubre **registrar + discovery + aprobar tool** (no el binding a agente, que es P02-S08).
- §11 (línea 165): "Ejecución automática de herramientas MCP de escritura sin aprobación humana queda prohibida" — relevante para P02-S08, no aplica al CRUD de esta slice; pero el contrato de defaults (`requires_approval=true`) debe sobrevivir el sync.
- §10 (línea 159): allowlist `MCP_ALLOWLIST_DOMAINS` env (§11.1 línea 540). En P02-S07-T001 — slice api/CRUD — la enforcement de allowlist puede vivir aquí (recomendado: validar `endpoint_url` contra allowlist en POST /servers; fail con 400 si no aplica) o diferirse a `policies.py` de runtime en P02-S08. **Decisión recomendada**: validar aquí, KISS — añade 10 líneas y previene registro de servidores no autorizados.

## Front → Back → DB contract

### Backend (esta slice)

- Router agregado: `app/mcp/__init__.py` exporta `mcp_router = APIRouter(tags=["admin-ai"])`. Se monta vía `app/admin/__init__.py` (incluir `mcp_router` en el `admin_router` aggregator) bajo prefix `/api/v1/admin/ai`. Paths internos al router: `/mcp/servers`, `/mcp/servers/{id}/sync`, `/mcp/tools/{id}`.
- **Decisión clave (D-WIRE)**: ¿modificar `app/admin/__init__.py` para incluir `mcp_router`, o modificar `app/main.py` directamente con un nuevo `include_router(mcp_router, prefix="/api/v1/admin/ai")`?
  - **Recomendación**: ampliar `app/admin/__init__.py` (un solo aggregator), **NO** tocar `main.py`. Esto **NO viola el write_set canónico** porque `backend/app/admin/__init__.py` quedaría fuera del declared write_set `backend/app/mcp/**` → necesita **WRITE_SET_DRIFT §D-MCPWIRE** explícito en el handoff, con justificación: "without registering the new mcp_router in the admin aggregator, the 4 endpoints are unreachable; reusing the existing admin_router pattern (P02-S05-T001) avoids touching main.py and keeps wiring DRY".
  - Alternativa rechazada: incluir directamente en `main.py` (precedente P02-S05-T001 ya usó `WRITE_SET_DRIFT §D-AAM`). Hace falta una de las dos; KISS dice un solo punto de wiring → admin aggregator.
- **Service layer**: `mcp.service.register_server(req, user) -> McpServer` orquesta:
  1. Validar transport ∈ {http, sse} + allowlist domain (si MCP_ALLOWLIST_DOMAINS está set y no contiene host del endpoint → 400).
  2. Crear `McpServer(status='draft', created_by=user.id)`.
  3. Si auth_type ≠ 'none': cifrar `auth.secret` con `encrypt_secret()` y crear `McpCredential(server_id, auth_type, encrypted_secret)`.
  4. Audit `admin.ai.mcp.server.create` con metadata `{server_id, name, transport, auth_type, allowlist_ok}` — sin secrets.
  5. Commit transaccional (server + credential atómico).
  6. Si EncryptionError → 500 + audit `admin.ai.mcp.server.create.failed` (D-S2 independent session).
- **Sync**: `mcp.service.sync_server(server_id, user) -> SyncResult` orquesta:
  1. Cargar McpServer + McpCredential (FOR UPDATE). Si server.id no existe → 404.
  2. Descifrar secret (si aplica) con `decrypt_secret()`.
  3. Llamar `mcp.client.discover_tools(server, credential) -> (tools, resources, prompts)`. Timeout configurable (default 10s).
     - Si transport error / timeout / non-2xx → 502 `MCP_SERVER_UNREACHABLE`, audit `admin.ai.mcp.server.sync.failed`.
  4. Upsert tools por `(server_id, name)`: si existe, **update no-destructivo** (description, input_schema, output_schema) preservando `enabled`, `requires_approval`, `risk_level`. Si no existe, insert con defaults (`enabled=false`, `requires_approval=true`, `risk_level='medium'`).
  5. **Decisión D-SYNC1 (idempotencia conservadora)**: no borrar tools que desaparezcan del servidor remoto en esta slice. Marcar como out-of-scope follow-up si producto lo requiere. **Justificación**: borrar puede invalidar bindings de agentes en P02-S08; safe-by-default.
  6. Upsert resources/prompts análogamente.
  7. `last_sync_at = now()`, `status = 'active'`.
  8. Audit `admin.ai.mcp.server.sync` con `{server_id, tools_count, resources_count, prompts_count}`.
  9. Return `{tools_count, status:'active'}`.
- **Patch tool**: `mcp.service.update_tool(tool_id, req, user) -> McpTool` orquesta:
  1. Cargar McpTool. Si no existe → 404.
  2. Validar (Pydantic): `enabled: bool?`, `requires_approval: bool?`, `risk_level: Literal['low','medium','high','critical']?`.
  3. Update only-provided-fields (PATCH semantics).
  4. Audit `admin.ai.mcp.tool.update` con `{tool_id, server_id, before:{enabled,risk_level,requires_approval}, after:{...}}`. **Audit `before` solo en metadata, no en logs estructurados**.

### Aceptación verificable (criterios concretos para validator/tester)

1. POST /servers cifra `auth.secret` con Fernet — `mcp_credentials.encrypted_secret` NUNCA contiene plaintext.
2. POST /servers acepta `transport ∈ {http, sse}` y rechaza `stdio` con 400 (Pydantic).
3. POST /servers/{id}/sync devuelve `{data:{tools_count,status:'active'}, meta:{request_id}}` y persiste tools con `enabled=false, requires_approval=true, risk_level='medium'` (DB defaults).
4. POST /servers/{id}/sync upsert no-destructivo: si tool `(server_id, name)` ya existía con `enabled=true`, el sync NO la desactiva ni cambia `risk_level`/`requires_approval`.
5. POST /servers/{id}/sync con server unreachable → 502 + envelope `{errors:[{code:'MCP_SERVER_UNREACHABLE'}]}` + audit row `admin.ai.mcp.server.sync.failed`.
6. GET /servers respeta admin role; employee → 403 `AUTH_PERMISSION_DENIED`; sin token → 401.
7. PATCH /tools/{id} valida `risk_level ∈ {low,medium,high,critical}` (default model dice 'medium' — incluir 'critical' por línea 183 del model: "low | medium | high | critical").
8. audit_logs filas creadas para las 3 acciones mutadoras (`admin.ai.mcp.server.create`, `admin.ai.mcp.server.sync`, `admin.ai.mcp.tool.update`) — D-S2 independent session.
9. Audit metadata NUNCA contiene `secret`, `password`, `token` (test invariante: grep substrings).
10. Logs BEFORE/AFTER por endpoint con `request_id`, sin secrets, dual verbose=true/false comprueba flow completo.
11. Rate limit POST /servers (10/min) y POST /sync (5/min) → 429 con `Retry-After` cuando se excede.
