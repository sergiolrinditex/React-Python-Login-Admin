## 1. Goal

Implement `POST /api/v1/admin/ai/providers/{provider_id}/discover-models` so that an Admin can hit a configured AI provider (Gemini-direct, OpenAI-direct, LiteLLM proxy), enumerate the models the provider currently exposes, reconcile against the `ai_models` table marking newly discovered rows with `auto_discovered=true`, and return a structured diff `{added, existing, skipped, total_seen}`. This is the runtime delivery of ADR-001 (LiteLLM dynamic model discovery), the contract that lets the model catalog stop drifting against vendor changes.

## 2. Acceptance criteria

Literal from work-item YAML + derived from ADR-001 + TECHNICAL_GUIDE §10.3 + non-negotiables:

A1. `POST /api/v1/admin/ai/providers/{provider_id}/discover-models` is implemented in `backend/app/features/admin_ai/`, mounted under the global `/api/v1` prefix, and wired into the FastAPI app router.
A2. Endpoint reconciles results against `ai_models` table (per ADR-001): existing rows by `(provider_id, model_id)` are LEFT UNCHANGED; new rows are INSERTED with `auto_discovered=true`, `enabled=false`, `is_default=false`, `model_type` derived from provider response (chat/embedding/other → "chat"|"embedding"|"unknown" mapped to the §10.3 `model_type` enum), `capabilities=[]` JSONB default, `pricing={}` JSONB default.
A3. Endpoint returns `{ data: { added: [AiModel], existing: [AiModel], skipped: [{model_id, reason}], total_seen: int } }` per the §6.2 envelope convention `{data, meta, errors}`.
A4. Endpoint is admin-only (returns 401 without auth, 403 with non-admin token). For P00 this is a stub guard — see §6.4.
A5. Endpoint returns 404 when `provider_id` does not exist; 502 with sanitized `{error.code, error.message}` when the upstream provider call fails (timeout, 4xx, 5xx); 422 for malformed `provider_id`.
A6. Provider credential is decrypted on-the-fly via `cryptography.Fernet` using `settings.encryption_key`; the cleartext credential lives only in the in-memory request scope and is NEVER logged, returned to the client, or persisted in plain form. The `ai_provider_credentials.encrypted_secret` column already stores Fernet-encrypted bytes (see §10.3 line 431 + T005 design notes).
B1. Provider-side dispatch supports the THREE provider types declared in `data/verification/admin_ai/providers.json`: `gemini` (direct, `GET {base_url}/v1beta/models?key={api_key}`), `openai` (direct, `GET {base_url}/v1/models` with `Authorization: Bearer {api_key}`), `litellm` (proxy, `GET {base_url}/v1/models` with `Authorization: Bearer {master_key}` — OpenAI-compatible shape). Other `provider_type` values return 422 with `error.code = unsupported_provider_type`.
B2. HTTP client is `httpx.AsyncClient` with explicit `timeout=httpx.Timeout(connect=5.0, read=15.0, write=5.0, pool=5.0)`, `transport=httpx.AsyncHTTPTransport(retries=0)` (no silent retries that would multiply rate-limit hits), and `verify=True` (TLS certificate verification on).
B3. Audit-log row is appended with `action="ai.provider.discover_models"`, `entity_type="ai_provider"`, `entity_id=provider_id`, `metadata={provider_type, total_seen, added_count, existing_count, skipped_count, request_id}`. **§15 ADR-001 — LiteLLM dynamic model discovery (literal)**

- **Fecha**: 2026-05-09
- **Contexto**: Los catálogos de modelos de los proveedores cambian continuamente; una lista de modelos hardcodeada en el bundle de seed deriva inevitablemente con el tiempo.
- **Decisión**: En runtime, `POST /api/v1/admin/ai/providers/{id}/discover-models` llamará al endpoint de modelos del proveedor y reconciliará los resultados contra la tabla `ai_models`, actualizando `auto_discovered=true` en las filas nuevas.
- **Alternativas descartadas**: (a) Hardcodear la lista de modelos por proveedor — rechazada: deriva en cuanto Google/OpenAI añade o retira modelos. (b) Exigir que el admin escriba el `model_id` manualmente — rechazada: UX hostil y propensa a errores tipográficos.
- **Consecuencias**: Round-trip adicional en la primera conexión; depende de que el proveedor exponga un endpoint compatible con la forma `/v1/models`. Detallado en FU-X1 (feature follow-up, P02-S05).

### 6.6 Riesgos concretos (max 5)

- **R1 — Gemini API contract drift**: `GET /v1beta/models?key=...` returns the array under `models` key with shape `{name, displayName, supportedGenerationMethods, ...}`; no `model_type` field. Mapping `supportedGenerationMethods ⊃ {"generateContent"}` → `chat`, `⊃ {"embedContent"}` → `embedding`, else `unknown`. Researcher should confirm the May-2026 response shape on shallow pass; if Google has migrated to `/v1/models`, prefer that path (newer API surface).
- **R2 — Provider key leak via base_url logging**: some providers embed the api_key in the URL query string (Gemini does: `?key=AIza...`). Logging the full URL would leak the key. Mitigation: log only `{provider_type, host, path}` — strip query string. Add a unit test that asserts `AIza`/`sk-` patterns never appear in log output for any verbose mode (T004 precedent).
- **R3 — Migration ordering vs P02-S01-T001**: when P02-S01-T001 lands, it will try to create `ai_providers/ai_models` again. Either P02-S01-T001 must be reshaped (split into `0004_chat_rag_mcp_agents.py` skipping admin_ai), or this slice's `0003` must be conservative enough that P02-S01-T001 can `IF NOT EXISTS` past it. Pack recommendation: this slice's migration uses raw `CREATE TABLE` (NOT `IF NOT EXISTS`); document in the handoff `## Source-of-truth amendments queued for closer` that P02-S01-T001's scope must be narrowed to `0004_chat_rag_mcp_agents.py`. Closer registers the source-of-truth amendment.
- **R4 — `auto_discovered` column not in §10.3**: The §10.3 DDL block does NOT declare `auto_discovered BOOLEAN`.
