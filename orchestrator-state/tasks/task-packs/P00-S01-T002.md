Minimum, production-quality skeleton (NOT a TODO):

- One `i18n` instance initialised in the same file or imported from a tiny inline init (since `i18n/index.ts` belongs to T005, T002 may inline a minimal `i18next.init({ resources: { en: { translation: {} } }, fallbackLng: 'en', lng: 'en', interpolation: { escapeValue: false } })`). - Exported component `Providers({ children }: { children: ReactNode })` that nests, from outside in: `BrowserRouter > QueryClientProvider > I18nextProvider > {children}`.
- BEFORE/AFTER logging in module init (see §7).

Function size: target ≤50 lines, file ≤200 lines. Single responsibility = wire providers.

These are real-behaviour tests (no mocked QueryClient, no mocked i18n; we are testing the actual providers). They satisfy `01-non-negotiables.md > Tests are REAL` because providers wiring IS the unit under test.

## 7. Logging contract

Per `01-non-negotiables.md > Logging`:

- Providers module init must log BEFORE/AFTER the `QueryClient` creation and the `i18n.init()` call.
- Verbose flag: Vite-style `import.meta.env.VITE_ENABLE_VERBOSE_LOGGING` (added to `.env.example` in T001 as `ENABLE_VERBOSE_LOGGING`; the frontend pulls the prefixed `VITE_*` variant; developer MUST keep both env keys consistent and document the mapping in the handoff).
- `false` → only `console.warn`/`console.error`. `true` → `console.info` shows full flow.
- **NO PII** logged: no tokens, no user objects, no emails. Provider init logs are purely structural ("providers.queryClient.created", "providers.i18n.initialised").
- No code path is modified to remove logs; only the flag changes.
