### Shape contract for `frontend/src/app/providers.tsx` (the only product code file)

The developer MUST export a component that downstream slices can rely on without re-discovery:

```tsx
// frontend/src/app/providers.tsx — SHAPE CONTRACT (planner-defined, do not deviate)
//
// Slice/Phase: P00 — Scaffold / P00-S01-T002 — Frontend dependency pack.
// Key deps beyond imports: see frontend/package.json (RHF, Zod, RR, RQ, i18next).
//
// Responsibility: single composition root for all React context providers used app-wide.
// MUST be safe to mount BEFORE the router is configured (T004 wires the router inside).
// MUST log init BEFORE and AFTER per 01-non-negotiables.md.

import type { ReactNode } from "react";

export interface ProvidersProps {
  children: ReactNode;
  // Optional injection seam so tests can pass a pre-built QueryClient and avoid
  // re-initialising the same singleton across renders.
  queryClient?: import("@tanstack/react-query").QueryClient;
  // Optional injection seam for the i18n instance. If omitted, providers must
  // import the default singleton — but T005 owns wiring real resources, so for
  // T002 the default can be a minimal pre-init i18next that exposes `t` returning
  // the key. Document this clearly in the file's docstring.
  i18n?: import("i18next").i18n;
}

export function Providers(props: ProvidersProps): JSX.Element;
```

Internally the composition order is fixed (outer → inner) so downstream slices know the assumption:

1. `<I18nextProvider i18n={i18n}>`  (T005 will replace the default with full resources)
2. `<QueryClientProvider client={queryClient}>` (P01-S03-T001 + all features rely on this)
3. `{children}`

`<BrowserRouter>` is NOT mounted here. T004 mounts the router as a CHILD of `<Providers>` or inside `App.tsx` between Providers and the route tree.
