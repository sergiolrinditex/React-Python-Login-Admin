## E. Endpoint contracts (front → back → DB)

### E.1 `GET /api/v1/admin/ai/agents` — list agents (admin)

- Auth: `require_admin` (people_admin or super_admin).
- Query params: none (V1). Future: `enabled` filter — out of scope.
- Response 200 envelope:
  ```json
  {
    "data": [
      {
        "id": "uuid",
        "name": "people_helper",
        "description": "Agente de ayuda...",
        "enabled": true,
        "config": { "model": "gpt-4o", "max_tokens": 1024, "mcp_server_ref": "..." },
        "bound_tools": [
          { "id": "uuid", "name": "list_policies", "server_name": "sandbox_readonly",
            "enabled": true, "requires_approval": false, "risk_level": "low" }
        ]
      }
    ],
    "meta": { "request_id": "<X-Request-ID>" }
  }
  ```
- Errors: 401 (no/invalid token), 403 (non-admin). No 404 — empty list returns
  `data: []` for the FE empty state.
- DB ops: SELECT agents + LEFT JOIN mcp_agent_bindings + JOIN mcp_tools (single
  query). No writes.

### E.2 `PATCH /api/v1/admin/ai/agents/{agent_id}/tools` — bind tools (admin)

- Auth: `require_admin`.
- Body: `{ "tool_ids": ["uuid", ...] }`. **Must be non-empty list?** No — empty
  list is the canonical "unbind all tools" operation. Validate type only.
- Validation chain (return 400 with envelope):
  1. `tool_ids` missing or not a list → 400 `AGENT_PAYLOAD_INVALID`.
  2. Agent not found → 404 `AGENT_NOT_FOUND`.
  3. Any tool id not found → 400 `AGENT_TOOL_NOT_FOUND` (include offending id).
  4. Any tool not approved (`enabled=false` OR not synced) → 400
     `AGENT_TOOL_NOT_APPROVED` (include offending id). Per invariant §C.8.
- Semantics: **set-replace** — the body declares the new full binding set.
  Implementation: DELETE existing bindings for `agent_id` not in `tool_ids`,
  INSERT new ones not in the existing set, leave overlap untouched. Inside one
  transaction.
- Response 200 envelope: same shape as the agent object in E.1 (one item, not
  wrapped in a list — `{"data": Agent}`).
- Audit row: `admin.agent.tools.update`, actor=admin user, entity=agent,
  metadata={request_id, ip, user_agent, tool_ids, added: [...], removed: [...]}.
- Error codes: 400, 401, 403, 404 per §C.4.

### E.3 `POST /api/v1/agents/runs` — start agent run (admin)

This is the **smoke** path. V1 keeps it deliberately minimal:

- Auth: `require_admin` (V1 — see §C.4 footnote on "Sí admin").
- Rate limit: `start_run_limiter` (Redis sliding window). Conservative
  default: 5 req/min/user. Fail-closed on Redis error (mirror MCP pattern).
- Body: `{ "agent_id": "uuid", "input": "string non-empty <= 4000 chars" }`.
  Validation: empty/missing → 400 `AGENT_RUN_PAYLOAD_INVALID`.
- Pre-flight:
  - Agent exists + `enabled=true` → else 404 `AGENT_NOT_FOUND` (404 even if
    `enabled=false`, to avoid leaking the disabled state to non-admins — but
    here caller is admin, so design choice: emit 409 `AGENT_DISABLED` with
    explicit message. **Confirm with researcher? No — invariant rule:** admins
    can see all state; emit 409 `AGENT_DISABLED` when found-but-disabled. Add
    code to TECHNICAL_GUIDE §6.4 error code list as part of this slice's
    proposed addendum (info-only — handoff RFC).
  - Bound tools enumerated; any binding with `tools.requires_approval=true`
    that does NOT yet have an `mcp_approvals` row with `status=approved` for
    this run → the run cannot include that tool. For V1 smoke: log + exclude;
    don't fail the run (see §F.5).
- Smoke execution:
  1. INSERT `agent_runs(status='pending', input, user_id, agent_id)` →
     `run_id`.
  2. Build DeepAgent from `deepagents_runtime.build_agent(agent, approved_tools,
     run_id, request_id)`.
  3. UPDATE `agent_runs.status='running'`.
  4. Execute **a single blocking smoke step** (NOT a full conversation). The
     simplest valid path: invoke the DeepAgent with `input` and capture the
     first response. For the smoke test, this can be one of two paths
     depending on what the researcher confirms:
     - **Path A (preferred)**: real DeepAgent.invoke() blocking call with the
       sandbox MCP server + the read-only `rag_tool`. No external LLM.
       Requires DeepAgents to accept a deterministic/mock LLM provider that
       does NOT hit a real model in CI.
     - **Path B (fallback)**: if DeepAgents requires a real LLM call to
       complete (i.e. cannot run without `langchain-anthropic` or
       `langchain-google-genai` credentials), the legitimate mock boundary
       per `01-non-negotiables.md` is the LLM HTTP client (third-party we do
       not control). DO NOT mock RAG, MCP, DB, or the DeepAgents tool layer.
     The researcher questions in §H pin which path applies. **Until §H
     answers, default to Path A and let the test mark `pytest.skip` only if
     DeepAgents 0.5.9 refuses to run without a real LLM provider.**
  5. UPDATE `agent_runs.status='done'`, `output=<truncated str>`,
     `finished_at=now()`.
  6. For every MCP tool the agent invoked, write a
     `mcp_tool_invocations(arguments_json, result_json, status='success',
     latency_ms, agent_run_id, tool_id)` row.
  7. On any exception inside steps 2–4: UPDATE `agent_runs.status='failed'`,
     populate `output` with a redacted error summary; raise to handler as 502
     `AGENT_RUN_FAILED` (per §C.4 error codes list).
- Response 200 envelope: `{"data": {"run_id": "uuid", "status": "done"}}`.
  (502 + envelope when failed.)
- Audit row: `admin.agent.run.start` on insert, `admin.agent.run.complete`
  on terminal status. Independent session (D-S2 pattern).
- Errors: 400 (payload), 401, 403, 404 (agent), 409 (disabled), 502 (external
  failure — MCP unreachable, DeepAgents failure).

### G.2 Required test cases (TCs)

1. **TC01 — list agents happy path**: admin GETs `/agents` after seeding one
   agent + one tool binding → 200, payload matches schema, `bound_tools`
   includes the bound tool with required_approval/risk_level/enabled.
2. **TC02 — list agents empty**: admin GETs `/agents` with no rows → 200,
   `data: []`.
3. **TC03 — list agents non-admin 403**: employee user → 403
   `AUTH_FORBIDDEN`.
4. **TC04 — list agents no auth 401**: no Bearer token → 401
   `AUTH_SESSION_EXPIRED` (reuse auth envelope).
5. **TC05 — bind tools happy path (set-replace)**: existing binding {A,B},
   PATCH with {B,C} → 200, final bindings {B,C}, audit row written.
6. **TC06 — bind tools to empty list (unbind all)**: existing bindings {A,B},
   PATCH with `tool_ids: []` → 200, final bindings {}, audit row written.
7. **TC07 — bind tools agent not found**: PATCH random UUID → 404
   `AGENT_NOT_FOUND`.
8. **TC08 — bind tools tool not found**: PATCH with one unknown tool id → 400
   `AGENT_TOOL_NOT_FOUND`, no partial write.
9. **TC09 — bind tools tool not approved**: PATCH with a tool that has
   `enabled=false` → 400 `AGENT_TOOL_NOT_APPROVED`, no partial write.
10. **TC10 — bind tools payload invalid**: missing `tool_ids` field → 400
    `AGENT_PAYLOAD_INVALID`.
11. **TC11 — start run happy path (smoke)**: admin POSTs
    `{agent_id: people_helper.id, input: "hello"}` → 200 with
    `{run_id, status: "done"}`, `agent_runs` row final state `status='done'`,
    `finished_at` set, audit rows for start + complete.
12. **TC12 — start run agent not found**: random UUID → 404 `AGENT_NOT_FOUND`.
13. **TC13 — start run agent disabled**: agent with `enabled=false` → 409
    `AGENT_DISABLED`. `agent_runs` row NOT created.
14. **TC14 — start run payload invalid**: empty `input` → 400
    `AGENT_RUN_PAYLOAD_INVALID`.
15. **TC15 — start run non-admin 403**: employee user → 403.
16. **TC16 — start run mcp unreachable (smoke fail path)**: simulate the MCP
    sandbox HTTP server returning a connection error → 502
    `MCP_SERVER_UNREACHABLE`. `agent_runs` row final state `status='failed'`,
    `output` contains redacted error summary, audit rows present. **This is
    the only legitimate mock**: the external HTTP MCP server is a
    third-party-style boundary per `01-non-negotiables.md` ("external
    third-party APIs you do not control"). Mock by monkeypatching
    `app.mcp.client` HTTP layer, NOT the DeepAgents runtime.
17. **TC17 — start run tool invocation audit**: after TC11 succeeds, assert
    `mcp_tool_invocations` has at least one row linked to the run_id with
    `arguments_json` populated and `status='success'` (or `denied` if the run
    excluded approval-required tools — see §E.3 step 5). If the smoke does
    not exercise any MCP tool, this TC asserts ZERO invocation rows AND
    documents that `people_helper` did not need MCP for this smoke; that is
    still a passing smoke because the DeepAgent ran and produced output.
18. **TC18 — both verbose modes**: full suite runs once with
    `ENABLE_VERBOSE_LOGGING=true` and once with `=false`, no assertion errors
    in either mode (mirror MCP/RAG slice convention).
