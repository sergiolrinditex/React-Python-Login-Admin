## 6. Front → Back → DB contract

### 6.1 GET /api/v1/admin/ai/agents (200/401/403/500)

Wire shape (mirror `AgentOut`):

```ts
interface Agent {
  id: string;                       // uuid
  name: string;
  description: string | null;
  enabled: boolean;
  config: Record<string, unknown>;  // mirrors JSONB; default {}
  bound_tools: BoundTool[];         // default []
}
interface BoundTool {
  id: string;                       // tool uuid
  name: string;
  server_name: string;
  enabled: boolean;                 // approval state from mcp_tools
  requires_approval: boolean;
  risk_level: string;               // open string ("low"|"medium"|"high"|...)
}
```

Envelope: `{ data: Agent[], meta: { request_id } }`. Errors: 401 → re-auth via `onAuthFailure`; 403 → permission_denied UI; 500 → error_network UI with retry.

### 6.2 PATCH /api/v1/admin/ai/agents/{agent_id}/tools (200/400/401/403/404)

Body: `{ tool_ids: string[] }` (set-replace; empty unbinds all). Error codes (from `router_agents.py`):

- 400 `AGENT_TOOL_NOT_FOUND` (`details.field = "tool_ids"`)
- 400 `AGENT_TOOL_NOT_APPROVED` (`details.field = "tool_ids"`) — tool exists but `enabled=false` in mcp_tools
- 404 `AGENT_NOT_FOUND`
- 500 `INTERNAL_ERROR`

Response: `{ data: Agent, meta: { request_id } }` — full Agent with refreshed `bound_tools`.

### 6.3 POST /api/v1/agents/runs (200/400/401/403/404/409/502/500)

Body: `{ agent_id: string, input: string }` (input 1..4000 chars; pydantic `extra="forbid"`). Error codes (from `router_runs.py`):

- 400 (pydantic validation) — `input` empty / >4000 chars / unknown field
- 404 `AGENT_NOT_FOUND`
- **409 `AGENT_DISABLED`** — agent.enabled=false. Treat in UI as `error_validation` "agent is disabled" near the run launcher (not a network error).
- **502 `AGENT_RUN_FAILED`** — DeepAgents/LangGraph downstream unreachable in dev sandbox. Map to `AgentsRunUnreachableError`; UI shows `errors:AGENT_RUN_FAILED` i18n with retry. **This is expected evidence in dev** (P-43 #2 pattern).
- 429/503 from `start_run_limiter` (5 req/min) — map to a friendly "Try again in a moment".

Response: `{ data: { run_id, status }, meta: { request_id } }` with `status ∈ {pending|running|done|failed|cancelled}`.

### 6.4 DB tables touched (read-only from the page side)

`agents`, `mcp_agent_bindings` (via PATCH), `agent_runs` + `mcp_tool_invocations` + `audit_logs` (via POST /runs; backend side-effects). Page does NOT touch DB directly. Cleanup SQL in §9.
