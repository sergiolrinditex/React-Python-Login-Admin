## Contract Observed — P01-S02-T009

Front → Back → DB contract as observed during tests and curl checks:

### Route / Provider / API Client
- Route: POST /api/v1/auth/2fa/enroll
- Provider: mfa_router (FastAPI APIRouter, prefix /2fa) mounted in auth_router at /api/v1/auth
- Wiring: backend/app/features/auth/routes.py includes mfa_router at prefix /2fa

### Endpoint Observed
- POST /api/v1/auth/2fa/enroll → 201 Created (happy path)
- POST /api/v1/auth/2fa/enroll → 401 AUTH_INVALID_CREDENTIALS (wrong password)
- POST /api/v1/auth/2fa/enroll → 401 AUTH_INVALID_CREDENTIALS (unknown email, same code)
- POST /api/v1/auth/2fa/enroll → 201 Created (idempotent second call — secret rotated)
- POST /api/v1/auth/2fa/enroll → 422 (missing password field — Pydantic validation)

### DTO / Schema
- Request: MfaEnrollRequest(email: EmailStr, password: SecretStr) — backend/app/features/auth/schemas.py
- Response: MfaEnrollResponse {data: MfaEnrollResponseData{otpauth_url: str, qr_png_base64: str}}
- Error: {errors: [{code: AUTH_INVALID_CREDENTIALS, message: str, field: null, details: []}]}

### Service (Use Case)
- enroll_mfa() in backend/app/features/auth/mfa_service.py
- Steps observed: re-auth → secret gen → encrypt → upsert → otpauth_url → qr_png → audit_log → commit

### Repository / DB Functions
- get_user_for_enroll() — queries users table by email
- upsert_mfa_secret() — INSERT or UPDATE mfa_totp_secrets
- insert_audit_log_2fa_enroll() — INSERT into audit_logs

### Tables Observed
- mfa_totp_secrets: user_id PK, secret_encrypted TEXT (140 chars Fernet token), enabled=false
- audit_logs: actor_user_id, action='auth.2fa_enroll', resource='POST /api/v1/auth/2fa/enroll', request_id, metadata JSONB
- users: queried for email lookup + password_hash verify

### Logging Contract Observed
- BEFORE/AFTER events: auth.mfa.enroll.start, auth.mfa.enroll.reauth_ok, auth.mfa.enroll.secret_generated, auth.mfa.enroll.persisted
- ERROR events: ERROR auth.mfa.enroll.invalid_credentials (warning level)
- CWE-532: NO secret, NO otpauth_url, NO qr_png_base64 in any log line
- email_masked format: t***@gmail.com (confirmed in back.log)
- ENABLE_VERBOSE_LOGGING=true: BEFORE/AFTER visible, no secret leakage
- ENABLE_VERBOSE_LOGGING=false: no INFO lines in enroll flow (test_t6 verified)
