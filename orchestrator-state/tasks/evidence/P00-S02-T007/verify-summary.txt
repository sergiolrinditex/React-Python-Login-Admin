== Findings ==
F1. (medium, in-scope spirit, write_set drift candidate) Wizard cannot reach FU-X1
    in localhost dev because frontend uses relative URL '/api/v1/...' and the Vite
    dev server has NO proxy entry for /api/v1. Result: every wizard request lands
    on Vite (which returns 404 with empty body and Vary: Origin), never on the
    backend at :8000. A3 acceptance ("Wizard POST against FU-X1 … renders response")
    is verified by Vitest mocks but BROKEN in real-browser dev. Production path
    (Docker / nginx) would proxy correctly via infra/nginx/default.conf, but the
    documented verify-slice command (open http://127.0.0.1:5174/admin/ai/models/new)
    cannot exercise the happy path.
    Files involved: frontend/vite.config.ts (no server.proxy block) and/or
                    frontend/src/features/admin_ai/data/discoverModels.ts
                    (API_BASE = '/api/v1' hardcoded, no env-var fallback).
    Recommended fix (smallest safe): add to vite.config.ts:
        server: {
          host: '0.0.0.0',
          port: Number(process.env.FRONT_PORT) || 5173,
          proxy: { '/api/v1': { target: 'http://127.0.0.1:8000', changeOrigin: true } },
        }
    Path-A semantic: dev-tooling change required to satisfy A3 in human-verify mode,
    symmetric to the existing T007 Path-A drift (main.tsx + 3 locale JSONs).

F2. (medium, pre-existing, NOT a T007 regression) admin_ai seed loader fails at

    Already tracked as FU-20260509220235 (admin_ai seed loader column). Verify-slice
    fell back to manual asyncpg insert. T007 closer should NOT be blocked on this.
