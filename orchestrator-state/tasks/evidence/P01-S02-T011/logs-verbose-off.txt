# ENABLE_VERBOSE_LOGGING=false verification

## Method
Used the running production backend on port :8000 (ENABLE_VERBOSE_LOGGING=false in root .env).
Measured back.log file size before and after a sign-up call.

## Result
- back.log size before POST /api/v1/auth/sign-up: 36914 bytes
- back.log size after POST /api/v1/auth/sign-up: 36914 bytes
- Delta: 0 bytes — no new log lines emitted

## Interpretation
With ENABLE_VERBOSE_LOGGING=false, the successful sign-up produces ZERO log output
to back.log (only WARNING+ level would appear). This is correct per non-negotiables
§Logging: "ENABLE_VERBOSE_LOGGING=false shows only warning+error".

## $argon2id$ check
Not applicable — no log lines were emitted. Trivially PASS:
if no lines are written, no $argon2id$ can be in those lines.
