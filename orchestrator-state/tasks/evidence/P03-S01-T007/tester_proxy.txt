NOTE: The 404 is expected — /api/v1/health is not a registered backend endpoint. The critical
observation is that the response comes from uvicorn (server: uvicorn header), not from Vite
itself (which would return a different error without server:uvicorn). Proxy confirmed working.

## Vite proxy test — /api/v1/auth/sign-in (POST-only endpoint)

STATUS: 405 from uvicorn — GET method not allowed for POST endpoint, but proxy IS forwarding.
The Allow: POST header confirms this is a real backend response. PASS.

## Proxy verdict
Both proxy checks return responses with "server: uvicorn" — confirming:
1. The Vite proxy IS forwarding /api/* to the backend at :8000.
2. Relative URL paths (e.g. /api/v1/users/me) will correctly reach the backend through Vite proxy.
3. The ADR-002 same-origin contract is enforced at the infrastructure level.
PROXY: PASS
