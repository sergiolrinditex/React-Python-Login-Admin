### Endpoints Verified
- POST /api/v1/auth/sign-in → 200 mfa_required:true + challenge token (or access_token on non-MFA)
- POST /api/v1/auth/2fa/verify → 200 + access_token + Set-Cookie refresh_token
- POST /api/v1/auth/refresh → 200 + new access_token + rotated cookie (Path=/api/v1/auth)
- GET /api/v1/users/me → 200 + UserProfile (all fields present, matching frontend domain types)
- POST /api/v1/auth/logout → 204 + Max-Age=0 cookie clear

### Tables Observed (read-only via backend)
- users: active record for employee.verification@inditex-sandbox.com
- refresh_tokens: rotated on /refresh, revoked on /logout (observed in audit logs)
- mfa_totp_secrets: active for employee.verification user
- audit_logs: auth.mfa.verify + auth.refresh + auth.logout entries
