## Decisions

### §D-T007-WRITE-SET-DRIFT-AUTHREPO
- Extended `authRepository.ts` with `updateLanguage(token, lang)` (drift option A).
- Justification: language endpoint is part of the auth/user contract; lives next to `fetchMe`/`logout`; moving the call to `presentation/` would violate Clean Architecture.
- Reference: `TECHNICAL_GUIDE §6.2`, same precedent as §D-T003-I18N / §D-T004-ROUTER / §D-T004-I18N-EXTEND.

### §D-T007-WRITE-SET-DRIFT-I18N
- Extended `i18n/index.ts` with `account` namespace keys in ES/EN/FR.
- Reference: same pattern as §D-T003-I18N, §D-T004-I18N-EXTEND.

### §D-T007-USERREAD-CHOICE
- Used `useAuth().user` directly for read (no separate `useMe` query). Lighter, fewer files; AuthProvider keeps user current on hydration.
- On successful PATCH, `confirmedCurrent` state (D1 fix) is set from server echo so the radiogroup reflects the server-confirmed language immediately without waiting for next hydration.

### §D-T007-D1-CONFIRMED-CURRENT (debugger cycle 1)
- Introduced `confirmedCurrent: Language | null` state in `useLanguagePicker`.
- Derived `current = confirmedCurrent ?? user.preferred_language ?? i18n.language`.
- Set only on PATCH 200 success from server echo (`result.value.preferred_language`).
- NOT touched on validation/network failure — radiogroup falls back to last server-confirmed value on error, not the stale optimistic in-flight value.

### §D-T007-D2-NETWORK-RETRY (debugger cycle 1)
- Replaced ad-hoc `<div role="status">` in AccountPage with `<NetworkErrorView onRetry={handleLanguageRetry} loading={isLanguagePending} />`.
- `handleLanguageRetry` re-invokes `setLanguage(currentLanguage)` — idempotent, uses last confirmed value.
- GET /users/me network error path still collapses into `permission_denied` via AuthProvider hydration — upstream of T007 write_set, accepted trade-off per §D-T007-PERMISSION-DENIED-NA.

### §D-T007-LANG-PICKER-VARIANT
- Radiogroup (not `<select>`). Keyboard-navigable, mobile-friendly, no platform dropdown chrome variance. Mirrors §D-T004-LANG-PICKER-VARIANT.

### §D-T007-LOGOUT-CONFIRM
- No confirmation modal. One click → logout. Defensive invariant (AuthProvider clears token even on 401/network error).

### §D-T007-NAVBAR-DEFERRED
- No navbar link added. Chat shell not in write_set. Verify-slice used direct URL `/account`. The acceptance bullet about "link en navbar" from original FU YAML is informational — not a blocking requirement for this slice. FU-20260518111311 registered for main-orchestrator triage.
