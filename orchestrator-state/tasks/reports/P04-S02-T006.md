## Executive Summary

This slice was promoted from follow-up `FU-20260516044404`, which reported that `POST /api/v1/admin/ai/mcp/servers/{id}/sync` returned HTTP 500 instead of HTTP 502 when the remote MCP server was unreachable. Investigation revealed that the production code has correctly mapped `McpServerUnreachableError → HTTP 502 + code=MCP_SERVER_UNREACHABLE` since commit `4e68680` (P02-S07-T001, `backend/app/mcp/router_servers.py:226-234`). The reported 500 was a runtime false positive caused by uvicorn PID 22130 serving stale bytecode from a deleted worktree (`P03-S01-T005/backend`) that predated the 502 mapping. The real delivery of this slice is the addition of five real-transport regression tests (TA-TE) in `backend/tests/integration/test_mcp_sync_unreachable.py` that exercise the full `httpx` transport path without mocking `client.discover`. These tests close the gap identified in the FU — the pre-existing T14 test mocked at the service boundary and could not catch a real-path transport regression. Verify-slice confirmed HTTP 502 + correct envelope via Chrome DevTools MCP (Plan B backend surface), real audit log row, and all three required ERROR log lines. No production code was modified.
