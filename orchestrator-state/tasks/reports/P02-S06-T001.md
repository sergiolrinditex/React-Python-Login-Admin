## Decisions

### Architecture

- **Subpackage split (§D-RAGDOCS-PKG):** `backend/app/rag/documents.py` (canonical write_set) expanded to 14-file subpackage. Rationale: 3 endpoints × (router + service) + repository + schemas + storage + audit + cursor + errors exceeded the 300 LoC cap for a single file. Pattern follows P02-S05-T001/P02-S07-T001/P02-S08-T001 precedent. Approved by validator cycle 1 and cycle 2.
  - Source ref: TECHNICAL_GUIDE §6.2; IMPLEMENTATION_CHECKLIST §Step 2.6; handoff §validator (cycle 1) File-size findings
- **Test split (§D-RAGDOCS-TESTSPLIT):** 3 test files instead of 1 (1151 LoC total). Per-endpoint split mirrors per-endpoint router split, maintaining 1:1 mapping.
  - Source ref: same as above
- **MIME validation strategy (Q2):** Hand-rolled magic bytes sniffer (4 bytes) + zipfile structural check (`word/document.xml` presence for DOCX) + `python-docx`/`pypdf` parse-attempt as second layer. No `python-magic` (OS dependency, Docker complexity, unnecessary). OWASP defense-in-depth pattern.
  - Source ref: official-docs-researcher Q2; TECHNICAL_GUIDE §6.2 security contract
- **Body-size enforcement (Q3):** Two-layer enforcement — layer 1: `Content-Length` header check in router before multipart parsing; layer 2: chunked `await file.read()` loop with byte counter in service. Starlette 0.49+ has no built-in `MaxRequestBodySizeMiddleware`.
  - Source ref: official-docs-researcher Q3
- **MinIO integration (Q4):** boto3 `put_object` (single-part, explicit `ContentType`) with `endpoint_url=S3_ENDPOINT_URL`. No `minio` Python SDK — consistency with worker `_storage._s3_client` contract. KMS/SSE not required for RELEASE.2025-09-07.
  - Source ref: official-docs-researcher Q4; TECHNICAL_GUIDE §3.3 storage env contract
- **Celery dispatch from async (Q5):** `await asyncio.to_thread(index_document_task.apply_async, args=[str(doc_id)])`. `apply_async` is blocking (Kombu sync TCP); wrapping in `asyncio.to_thread` avoids blocking the event loop.
  - Source ref: official-docs-researcher Q5
- **Storage order for MinIO/DB atomicity (R8):** `documents` row INSERT + commit → MinIO `put_object` → on MinIO failure: DELETE the row + commit. Documented in `service_upload.py` docstring.
  - Source ref: task pack §Developer R8 Decision
- **python-multipart pin (§D-RAGDOCS-DEPS):** `python-multipart==0.0.28` is NOT a transitive dependency of `fastapi==0.135.2`. Must be explicitly pinned; FastAPI 0.60.2+ raises `ImportError` eagerly at startup if `UploadFile`/`Form()` are used without it.
  - Source ref: official-docs-researcher Q1 RESOLVED; TECHNICAL_GUIDE §5 (dependencies)

### Debugger cycle 1

- **Dead code removal:** `repository.create_document` (~50 LoC) was defined but never called (service_upload.py constructs `Document(...)` directly). Deleted per YAGNI + DRY validator finding. `repository.py` LoC: 383 → 327. Zero behavioral regressions (all 29 tests still green).
