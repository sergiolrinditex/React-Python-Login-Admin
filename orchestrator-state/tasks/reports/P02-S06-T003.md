## Decisions

| Decision | Source reference |
|----------|-----------------|
| POSIX sh only (`#!/bin/sh`, `set -eu`, no `set -o pipefail`, `[ ]` not `[[ ]]`, no `local`, no arrays) | Task pack §3 + §7.1 constraint: compose entrypoint forces `/bin/sh` on Alpine-based `minio/mc:latest` image |
| `eval`-based `require_env()` for portable env validation | POSIX sh has no `${!varname}` indirect expansion; `eval "val=\${$1:-}"` is the canonical portable alternative; constraint names are hardcoded — no user-controlled input reaches eval |
| Retry 5×2s around `mc alias set` | Task pack §9.5 R4: S3 API warm-up race; conservative defensive margin vs researcher Q3 recommendation of 3×2s |
| `mc alias set local` — alias name `local` | Ephemeral inside one-shot container; collision-free with host; short and readable |
| Endpoint `http://minio:9000` hardcoded | Task pack §7.1.3 explicitly mandates non-configurable; avoids over-engineering; change travels with compose service name change in same diff |
| `mc alias set` stdout+stderr suppressed (`>/dev/null 2>&1`) | `01-non-negotiables.md §Security`: credentials must not appear in logs; security tradeoff: creds protection over verbose diagnostics |
| `mc mb --ignore-existing` for idempotency | Task pack §2 acceptance literal; stable since MinIO mc RELEASE.2022+; canonical idempotency idiom |
| Logs on stdout with `==> ` prefix | Matches project convention in `dev-restart.profile.sh` and `setup-from-scratch.sh` |
