## Decisions

- **D-T003-FIRST-PAGE-ONLY**: First page only; no infinite scroll for v1. `has_more` is returned in the response but ignored by the UI per task pack §2.4. Reference: `HILO_PEOPLE_TECHNICAL_GUIDE §6.4`.
- **D-T003-SPLIT-ERRORVIEWS**: HistoryPage.tsx pre-split reached 409 raw / 348 non-blank lines. Proactively extracted LoadingSkeleton, EmptyState, NetworkErrorView, ForbiddenView to `_HistoryPage.error-views.tsx`. Final sizes: HistoryPage.tsx 272 raw / 228 substantive, _HistoryPage.error-views.tsx 199 raw — both under 300-line cap per `01-non-negotiables.md §File size`.
- **D-T003-CHEVRON-TEXT**: Used `›` Unicode character directly in JSX (not `&#8250;` HTML entity; scanner detects `#8250` as hex literal). UX_CONTRACT §8 explicitly accepts text-rendered chevron characters.
- **D-T003-HOOK-RETRY**: Removed `retry: 1` from useHistory hook. Retry behavior is controlled by QueryClient `defaultOptions` so tests can set `retry: false` per the pattern from T001/T002. Reference: `HILO_PEOPLE_TECHNICAL_GUIDE §6`.
- **D-T003-GROUPING-NOON**: `daysAgo` helper in tests uses `setHours(12,0,0,0)` to avoid midnight boundary ambiguity in `startOfDay` computation.
- **D-T003-I18N-NAMESPACE-COUNT**: I18N_NAMESPACES went from 8 to 9. Pre-existing `i18n.test.ts` assertion updated accordingly (WRITE_SET_DRIFT pre-authorized §D-T003-I18N in task pack §5).

## Debugger cycles

### Cycle 1 — MCP artifact, no product code change

- **Root cause**: The slice-verifier MCP session was hitting the wrong Vite dev server. The P03-S02-T002 worktree's Vite process (PID 79038) held port 5173 and had no `/history` route; requests hit the catch-all `*` which redirected to `/chat`.
- **Evidence**: `curl http://localhost:5173/src/app/router.tsx | grep _jsxFileName` returned the T002 worktree path; `lsof -i tcp:5173 -sTCP:LISTEN` confirmed PID 79038 = T002 Vite.
- **Files changed**: none (product code was correct as shipped).
- **Remediation** (orchestrator-level, not a product follow-up per `05-runtime-write-contract.md`): T002 Vite was killed and T003 Vite was restarted on port 5173, then `/verify-slice` was re-run. This was mechanical orchestration noise — not a code defect, not a follow-up candidate.
- **Evidence**: `orchestrator-state/tasks/evidence/P03-S02-T003/debug-summary.md`, `debug-history-redirect-to-chat.png`, `debug-history-renders-correctly-on-t003-vite.png`, `debug-served-router-source.txt`, `debug-vite-process-on-5173.txt`

### Cycle 2 — EmptyState body paragraph fix (in-scope code omission)

- **Root cause**: `EmptyState` in `_HistoryPage.error-views.tsx` rendered Wordmark → TrackedLabel(`empty.title`) → SolidCTA, omitting the body paragraph required by task pack §4 ("body explaining 'starting a new chat' (i18n key `history.empty.body`)") and UX_CONTRACT §4. The i18n key was populated in all 3 locale files and 3 inline resources but never consumed in JSX.
- **Fix**: Smallest safe diff inside pre-authorized `§D-T003-PAGE-SPLIT-ERRORVIEWS` write set: added `EMPTY_BODY_STYLE: CSSProperties` constant (tokens-only: `var(--font-sans)`, `var(--color-ink)`, `opacity 0.85`, `textAlign: center`) and a single `<p style={EMPTY_BODY_STYLE} data-testid="history-empty-body">{t("history:empty.body")}</p>` between TrackedLabel and SolidCTA.
- **Files changed**: `frontend/src/pages/chat/_HistoryPage.error-views.tsx` (+9 lines), `frontend/src/pages/chat/__tests__/HistoryPage.test.tsx` (+6 lines, P02 extended to assert body present + ES productive copy).
