## Decisions

| Decision | Description | Source |
|----------|-------------|--------|
| §D-T002-AUTH-PORT | Extend IAuthRepository port with SignUpRequest + SignUpOutcome + signUp(). Mirrors T001 signIn() pattern. Domain layer has zero external imports. | `TECHNICAL_GUIDE §4` / Clean Architecture |
| §D-T002-AUTH-DATA | `AuthRepository.signUp()` uses plain `fetch()` (not `authFetch`) for the public endpoint. Mirrors signIn() shape: `credentials:'include'`, `X-Request-ID`, full error map per status code. | `TECHNICAL_GUIDE §5.1` |
| §D-T002-AUTH-ERRORS | 7 typed sign-up error classes. Naming follows `signup` lowerCamel convention (differs from `signin` for prior classes). | Clean Architecture error taxonomy |
| §D-T002-EMAIL-CORP | Client-side: zod `.email()` (RFC 5322 syntax only). Domain allowlist is server-authoritative (env-var `CORPORATE_EMAIL_DOMAINS`). Frontend surfaces 400 `AUTH_SIGNUP_NON_CORPORATE_EMAIL` as form-level `permission_denied` error. | `instrucciones.md §3.2` |
| §D-T002-PASSWORD-PRE-VALIDATE | Mirror server policy (min 12, ≥1 letter, ≥1 digit) in zod for UX. Server remains the final authority; client pre-check is a UX hint only. | `TECHNICAL_GUIDE §5.1` |
| §D-T002-LEGAL-LITERAL-TRUE | `z.boolean().refine(v => v === true, ...)` for legal_acceptance so unchecked checkbox shows field error before submit. | `UX_CONTRACT.md §3` |
| §D-T002-CHECKBOX-INLINE | Inline editorial checkbox in SignUpPage.tsx with design tokens only. No new shared component (YAGNI until 2nd page needs it). | `01-non-negotiables.md §YAGNI` |
| §D-T002-SUCCESS-REDIRECT | success → navigate(ROUTE_AUTH_SIGN_IN, { replace: true, state: { flash: "account_created" } }). NOT auto-login (sign-up response has no access_token). | `instrucciones.md §3.2` |
| §D-T002-409-NO-FIELD | On 409 EMAIL_TAKEN, do NOT highlight the email field (anti-enumeration). Server returns field=null intentionally. Generic copy shown. | `01-non-negotiables.md §Security` |
| §D-T002-PII-LOGGING | BEFORE: email_domain only, password_len numeric, has_full_name boolean, legal_accepted boolean. AFTER (success): user_id UUID only. NEVER log full email, password, full_name. | `01-non-negotiables.md §Logging` |
| §D-T002-PROACTIVE-EXTRACT | signUpHelpers.ts extracted to keep SignUpPage.tsx ≤300 eff. LoC (measured: 295 eff. lines). Applies T001 validator REC-1 preemptively. | `01-non-negotiables.md §File size` |
| ADR-002 debugger fix | `authRepository.ts:49` fallback changed from `"http://localhost:8000"` to `""`. Empty string produces relative paths `/api/v1/...` which route through Vite proxy (dev) and Nginx (prod) per ADR-002. Pre-fix: all browser submits hit port 8000 directly → CORS preflight failures for every submit path. | `TECHNICAL_GUIDE ADR-002` |
