| Decision | Rationale | Source doc ref |
|----------|-----------|----------------|
| `--check` mode added to `scripts/setup-from-scratch.sh` | Without it, `alembic upgrade head` would fail on a fresh checkout with no deps installed. `--check` skips DB/seed commands and exits 0 for structural-only acceptance. | TECHNICAL_GUIDE §3 commands; task pack §Risks #1 |
| `API_PORT` / `FRONT_PORT` (not `BACKEND_PORT` / `FRONTEND_PORT`) | These are the canonical env-var names read by `scripts/dev-restart.profile.sh`. Using the names the runtime actually reads avoids silent divergence. | STACK_PROFILE.yaml, dev-restart.profile.sh |
| `FRONT_PORT=5173` (not 5000) | Vite default port; `dev-restart.profile.sh` comment references 5000 but that is a Flutter-template leftover. 5173 aligns with the declared React/Vite stack. | TECHNICAL_GUIDE §6; task pack §Impact analysis #6 |
| No `backend/__init__.py` at repo root | Task pack explicitly states "prefer NO root `__init__.py`". Only `backend/app/__init__.py` is needed for `uvicorn app.main:app` resolution. | Task pack §Implementation notes #5 |
| stdlib `logging.basicConfig` in main.py (not structlog yet) | structlog lands in T003. For the scaffold slice, stdlib logging respects the `ENABLE_VERBOSE_LOGGING` contract without introducing external deps. | 01-non-negotiables.md §Logging; task pack §Implementation notes #6 |
| All deps DECLARED-ONLY at T001 | No `pip install` / `npm install` in this slice.
