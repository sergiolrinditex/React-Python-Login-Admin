| Decision | Description | Source ref |
|----------|-------------|------------|
| **D-T001-RBAC-SCOPE** | Audit route wired in its own `RequireRole(["people_auditor","super_admin"])` block — NOT inside existing `people_admin|super_admin` block. Backend `require_auditor` does not accept `people_admin`. | `instrucciones.md §3.2`, `TECHNICAL_GUIDE §6.1` |
| **D-T001-PAGINATION** | `allRows` state accumulates; `useRef(previousData)` distinguishes fresh query vs. load-more append; `appliedCursor` controls current page cursor. | `TECHNICAL_GUIDE §6.1` |
| **D-T001-PII-CLEAN** | BEFORE log: `{has_actor_present, action_present}` — never full UUID. AFTER log: `{count, has_more, request_id}` — never full rows. | `instrucciones.md §Security`, `01-non-negotiables.md §Logging` |
| **D-T001-METADATA-RENDER** | `renderMetadata` shows only `request_id` (first 8 chars) or "—". IP, user_agent, full metadata never reach DOM. | `TECHNICAL_GUIDE §6.1`, `UX_CONTRACT.md` |
| **D-T001-COMPONENT-SPLIT** (debugger) | AuditLogPage.tsx was 539 LOC with 3 components. Debugger extracted `_AuditTable.tsx` (121 LOC) and `_AuditFilters.tsx` (160 LOC), reducing page to 294 LOC.
