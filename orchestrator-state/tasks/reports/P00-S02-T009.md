## Objective

Suppress the `httpx` and `httpcore` stdlib loggers at `WARNING` to prevent Gemini API keys embedded in outbound request URLs (`?key=AIza...`) from leaking into verbose log output (CWE-532 third-layer defense). This is a pure backend infra slice with no UI surface, no endpoint change, and no DB migration.

## Decisions

| Decision | Rationale | Source |
|----------|-----------|--------|
| Named-logger approach (`logging.getLogger("httpx").setLevel(WARNING)`) chosen over event_hooks / Filter / custom transport | Smallest possible change (~3 functional lines), no write_set widening, no new deps, idempotent inside the existing `_configured` guard | Task pack §9.2 rejected alternatives |
| Applied UNCONDITIONALLY — not inside `if verbose:` branch | Defense-in-depth: guards against future commits that raise root level back to DEBUG without re-pinning httpx. Named pin is cheaper than cultural discipline. | `01-non-negotiables.md §Security` |
| Both `httpx` AND `httpcore` pinned to WARNING | httpcore is the transport layer that could surface DEBUG-level connection/header lines in future httpx versions (risk R1). Researcher confirmed canonical stdlib logger names for httpx 0.28.1. | Official-doc-notes file `P00-S02-T009-httpx-logger-leak-2026-05-10.md` (RESOLVED) |
| `httpx.MockTransport` used for T1/T3/T4 | The leak vector is the REQUEST URL logged by httpx **before** the transport runs. A fake transport still exercises the exact logging code path. Google Gemini is an external API; mocking is explicitly permitted per `01-non-negotiables.md §Tests are REAL`. | Handoff §Developer run decision #4 |
| `async def test_*` for async httpx tests | `httpx.AsyncClient` requires an async context; `pytest-asyncio asyncio_mode=auto` handles this automatically — same pattern as `test_admin_ai_discover_models.py`. | Handoff §Developer run decision #5 |
| `_restore_logging_state_with_httpx()` extends the T004 helper | Without it, T009 tests would leave httpx/httpcore loggers at WARNING in subsequent tests that check default levels, causing cross-test pollution. | Handoff §Developer run decision #6 |
