## Executive Summary

This slice implements the MCP server registration wizard at `/admin/ai/mcp/new`, the second screen of Journey J105. The wizard allows a People Tech Admin to register an external MCP server by entering transport, endpoint, auth type and (conditionally) credentials. The implementation extends the existing `features/mcp` module with new domain types, a repository method, a custom hook, and a new wizard page decomposed into four helper files to respect the ~300-line file-size cap. The backend endpoint (`POST /api/v1/admin/ai/mcp/servers`) was already implemented in P02-S07-T001; this slice adds only the frontend wiring.

The secret/credential path meets the §D-T004-SECRET-NEVER-PERSISTED invariant: the secret is captured as `type="password"` in RHF local state, transmitted via `authFetch` body, and never logged, echoed, or persisted in any client storage.
