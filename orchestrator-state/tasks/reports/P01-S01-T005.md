## Decisions

### WRITE_SET_DRIFT — 0003 down_revision updated (APPROVED IN-SLICE)

**Context**: Migration 0003 was committed (P00-S02-T006, SHA `0c2ea37`) with `down_revision = "0001"`. Inserting migration 0002 (a child of 0001) would create two heads (0002 and 0003 both from 0001), causing `alembic upgrade head` to fail with "Multiple head revisions are present".

**Decision**: Update 0003's `down_revision` from `"0001"` to `"0002"` (1-line metadata change, zero DDL impact). This is the only correct mechanical fix — `alembic merge` would add a synthetic 4th file with no semantic value.

**Validator verdict**: APPROVED IN-SLICE (5 reasons documented in handoff §WRITE_SET_DRIFT verdict). Planner gap, not developer error — 0003's original docstring explicitly stated it reserved slot 0002.

**Source ref**: `TECHNICAL_GUIDE §10.3`; validator memory pattern "honest developer flag earns ratification credit".

**Follow-up created**: `orchestrator-state/tasks/follow-ups/FU-20260510083509-planner-write-set-should-include-sibling-migrati.yaml` (severity=low, non-blocking).

### asyncpg INET decode returns ipaddress.IPv4Address (not str)

**Context**: Task-pack assumed asyncpg returns INET as `str`. Official-doc-research (2026-05-10) found asyncpg 0.31.0 `inet_decode` returns `ipaddress.IPv4Address` objects for plain host addresses when `native_inet_types` is not set to `False` (the project default).

**Decision**: Keep `Mapped[str | None]` ORM annotation (runtime-safe, SQLAlchemy does not enforce annotations). All test assertions use `str(row.ip)` for safe comparison. Do NOT add `native_inet_types=False` to `db.py` (outside T005 write-set).

**Source ref**: asyncpg 0.31.0 `network.pyx` `inet_decode`; SQLAlchemy asyncpg dialect `_disable_asyncpg_inet_codecs`; official-doc-note: `orchestrator-state/memory/official-doc-notes/2026-05-10-P01-S01-T005-asyncpg-inet-decode-ipaddress-not-str.md` (RESOLVED).

### PG18 INET information_schema type

`information_schema.columns` reports `data_type='inet'` for INET in PostgreSQL 18 (not `'USER-DEFINED'` as older versions may). Test 1 accepts both: `data_type == 'inet' OR udt_name == 'inet'`. Ref: `TECHNICAL_GUIDE §10.3`.

### Debugger fix — verbose logging compliance (cycle 1/3)

**Root cause**: `test_audit_logs_compliance_cols.py` used `structlog.get_logger(__name__)` at module top without calling `configure_logging()`. pytest does NOT import `app.main` (the production wiring point), so structlog used its default no-level-gate config, leaking INFO lines under `ENABLE_VERBOSE_LOGGING=false`.

**Fix (Option B)**: Bootstrap `configure_logging()` at module import inside the test file, using the same env-var read pattern as `app/main.py:40-44`. Did NOT use Option A (autouse conftest fixture) because `test_auth_signup.py:T8` deliberately resets `_configured` mid-suite — a global autouse would race with that. The `configure_logging()` is idempotent via `_configured` flag.

**Result**: verbose=false → 0 structlog INFO lines; verbose=true → BEFORE/AFTER INFO visible. 4/4 tests still pass, 15+4 regression tests unchanged.

**Rule ref**: `01-non-negotiables.md §Logging` — "ENABLE_VERBOSE_LOGGING=false → only warning+error".
