## Open Items

### VF-T004-1 — Cookie Path mismatch (system-level, NOT T004 defect)

- **Finding**: Sign-in helper `_set_refresh_cookie` sets `path="/auth"`. Actual endpoints live under `/api/v1/auth/*`. Per RFC 6265 §5.4, the browser will NOT send the cookie back to `/api/v1/auth/logout` because `/api/v1/auth/...` is not a subpath of `/auth/`. A real browser or `curl -b` cannot transmit the cookie automatically.
- **Classification**: `scope_expansion` — fix requires touching T002 sign-in setter + T003 refresh consumer + T004 logout consumer + TECHNICAL_GUIDE §10.2 in a coordinated change.
- **Resolution**: FU `FU-20260512082956-fix-refresh-cookie-path-mismatch-auth-vs-api-v1-` registered at severity=high and PROMOTED to DAG task **P01-S02-T011** (status=blocked, depends_on=P01-S02-T004) via explicit human approval 2026-05-12T08:30:00+00:00.
