**Root cause fixed**: When `check_handoff_contract.py` was invoked from a cwd outside the workspace root (e.g. from `/tmp` with only `CLAUDE_ORCHESTRATOR_ROOT` set and no `CLAUDE_WORKTREE_ROOT`), `handoff_path(task_id)` resolved relative to the workspace root which was not found from that cwd, returning `EXIT=2 "missing handoff"` even though the handoff existed in the canonical root. The fix adds `_resolve_handoff_path(task_id)` implementing an A→B candidate resolution (workspace first, canonical fallback) so the correct path is always found regardless of invocation cwd.
