## Decisions

### §D-T007-FIX-MIRROR-T002
Changed `const API_BASE = import.meta.env.VITE_API_BASE_URL ?? "http://localhost:8000"` to `?? ""`. Mirrors the identical fix applied to `authRepository.ts:49` in P03-S01-T002 (PR #28, commit `0a7d14c`). Rationale: ADR-002 same-origin reverse-proxy contract — all `authFetch` calls must use relative paths; `VITE_API_BASE_URL=""` is the canonical pin in `.env.example`.

Reference: `docs/source-of-truth/HILO_PEOPLE_TECHNICAL_GUIDE.md §ADR-002` + `§11.4 Cross-origin / reverse-proxy topology`.

### §D-T007-DOCSTRING-ADR
Extended the file docstring with an ADR-002 paragraph explaining that `API_BASE=""` makes all fetch calls relative, that Vite proxies `/api → :8000` in dev and Nginx proxies in prod, and that setting `VITE_API_BASE_URL=http://localhost:8000` is explicitly forbidden.

Reference: `docs/source-of-truth/HILO_PEOPLE_TECHNICAL_GUIDE.md §ADR-002`.

### §D-T007-COVERAGE-NEW-TESTFILE (WRITE_SET_DRIFT, pre-authorized)
