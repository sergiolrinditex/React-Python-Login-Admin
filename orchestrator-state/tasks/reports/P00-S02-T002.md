## Decisions

| Decision | Rationale | Reference |
|----------|-----------|-----------|
| D1 — flat shape `{status, version, uptime}` on /health and /live | Preserves T001 Docker compose healthcheck backward-compat; avoids breaking existing `curl -fsS http://localhost:8000/health` scripts | `HILO_PEOPLE_TECHNICAL_GUIDE.md §6.2` |
| D2 — redis/litellm checks declared `not_implemented` (not faked as "ok") | Forward-compatible shape for P02-S04-T002 (Celery) and future providers; avoids false-positive readiness reports | `instrucciones.md §3.3#observability` |
| D3 — `@app.middleware("http")` form for request_id middleware | FastAPI-idiomatic; avoids Starlette `BaseHTTPMiddleware.dispatch` `Response/Awaitable` mypy type mismatch; confirmed in official-docs-researcher note | Official doc note `P00-S02-T002-health-endpoints-patterns-2026-05-09.md` |
| D4 — `exc_info=True` removed from `_probe_db()` error branches | CWE-532 fix: structlog `ConsoleRenderer` + `RichTracebackFormatter(show_locals=True)` renders asyncpg/SQLAlchemy frame locals (cparams = {host, user, password}) to stdout in verbose mode. Structured fields `error_class` + sanitized `db_detail` remain. | `HILO_PEOPLE_TECHNICAL_GUIDE.md §8` + debugger memory |
| D5 — `_sanitize_db_error()` strips DSN components from error message before returning | Defense in depth — ensures response body never contains connection credentials even if error message leaks | `01-non-negotiables.md §Security` |
| D6 — regression test #9 as CWE-532 behavioral guard | Fake engine builds and raises exception with secret cparams as Python frame locals; test FAILS if `exc_info=True` is reintroduced | `01-non-negotiables.md §Tests are REAL` |
