## Decisions

| ID | Decision | Source |
|---|---|---|
| D-PAG1 | Cursor = base64url("<updated_at_iso>\|<uuid>"), limit+1 algorithm, next_cursor from last returned row. Index `conversations_user_updated_idx` covers WHERE+ORDER. | Task pack §D |
| D-TIT1 | initial_message present → title = first 60 chars (strip + "..."); absent → title = "" | Task pack §D |
| D-LANG1 | explicit language in request body → use it; absent → `current_user.preferred_language` | Task pack §D |
| D-TX1 | POST atomicity: flush conv row first (get id), insert message, UPDATE conversations SET updated_at=now(). `session.commit()` after service returns (not `session.begin()` — anti-pattern with FastAPI autocommit-False sessions). | Task pack §D; PROGRESS.md P-21 |
| D-RL1 | No rate limit on chat endpoints in T001. Redis limiter from P02-S02-T001 available but not wired. Deferred to hardening. | Task pack §D |
| D-AUD1 | Chat CRUD is NOT an auditable action per §Security/Audit log list. | `01-non-negotiables.md §Security` |
| D-LOG1 | BEFORE/AFTER per endpoint/service/repository with user_id_hash (SHA-256[:16]), request_id, route, latency_ms, status_code. Content never logged. | `01-non-negotiables.md §Logging` |
| D-option-b | Local `ChatResponseMeta` to avoid WRITE_SET_DRIFT on `app/auth/schemas.py`. | Task pack §F.2 |
| F-1 fix (option A) | `_CHAT_INVALID_PAYLOAD_PATHS` + union + `_invalid_payload_code_for_path` helper in `main.py`. Auth paths keep `AUTH_INVALID_PAYLOAD`; chat paths emit `CHAT_INVALID_PAYLOAD`. One handler body (DRY/KISS). | Validator cycle 1 §F-1; `01-non-negotiables.md §API contract` |
