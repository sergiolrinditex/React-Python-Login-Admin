## Executive Summary

This slice implements the AccountPage (`/account`) — the employee profile read-only view, language selector (ES/EN/FR), and logout flow. ## Security Notes

- Language PATCH input strictly typed `LanguageCode = 'es'|'en'|'fr'` on client; no arbitrary string can reach the wire.
- Logout: `useAuth().logout()` → `POST /auth/logout` + in-memory token cleared + React Query cache cleared + status=unauthenticated.
- 401 path: aggregated by `authFetch` refresh interceptor; final 401 → `UserAuthExpiredError` → `onAuthFailure` → `useAuth().logout()` → redirect.
- No tokens, emails, full_names, or PII in logs (`logger.ts:11-16`; `userRepository.ts:17-18`).
- No backend changes (no SQL/CORS/headers/permissions/infra).

## Accessibility Notes

- `role=radiogroup` on language picker container; `role=radio + aria-checked` per button; `aria-label` from i18n.
- `aria-busy="true"` on loading skeleton.
- Error containers: `role="alert"` (inline validation error) and `role="status" + aria-live="assertive"` (network/permission error views).
- `minHeight:"44px"` on all interactive targets (language buttons, logout button).
- Active language state: bullet character (●/○) + TrackedLabel text — not color alone (WCAG AA).
- Keyboard: native `<button>` elements, no custom focus trapping.

## Performance Notes

- `useMe` staleTime 60s with `initialData` from `useAuth().user` — avoids redundant GET /users/me on mount for already-authenticated sessions.
- Language mutation: optimistic UI update so the user sees the new locale instantly; no spinner on a successful tap.
- `lastIntendedLanguageRef` race guard prevents stale revert on rapid successive clicks.
