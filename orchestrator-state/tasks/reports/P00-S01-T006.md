## Slice summary

This slice originated from runtime follow-up `FU-20260516020115`, which tracked a key mismatch between `.env.example` (containing the stale value `sk-litellm-dev-only`) and the canonical LiteLLM bearer fixture `data/verification/admin_ai/credentials/litellm_sandbox_bearer.json` (containing `credential_plain=hilo-dev-litellm-master-key-2026`). Because `docker-compose.yml:92` propagates `LITELLM_MASTER_KEY` from the env to the LiteLLM proxy container, any developer doing `cp .env.example .env` would immediately get HTTP 401 on every `POST /chat/completions` call — the backend was sending the wrong bearer to the proxy. This slice corrects the value in `.env.example` (D1), adds the corresponding documentation row to TECHNICAL_GUIDE §11.1 (D2), and extends the in-file comment in `.env.example` to reference the fixture dependency (D3). No product code was changed. The fix propagates to canonical `main` on PR merge.
