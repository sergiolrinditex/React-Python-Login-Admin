## Decisions

### D1 — Cast form: `CAST(:meta AS JSONB)` over `(:meta)::jsonb`
Reference: `HILO_PEOPLE_TECHNICAL_GUIDE.md §10.3` (employee_profiles DDL), `01-non-negotiables.md §Security` (parametrized queries).
Rationale: SQLAlchemy 2.x `text()` named-parameter parser treats `::` after a bind parameter as ambiguous. `CAST(:meta AS JSONB)` is SQL-standard and unambiguous. Applied uniformly at all six cast sites for consistency.

### D2 — Serializer: `json.dumps()` over `str(dict).replace("'", '"')`
Reference: `01-non-negotiables.md §Error handling`, `HILO_PEOPLE_IMPLEMENTATION_CHECKLIST.md#Runtime Follow-up Coverage Registry`.
Rationale: `str(dict).replace("'", '"')` is a fragile Python-repr hack that corrupts booleans (`True`/`False` vs `true`/`false`) and null values (`None` vs `null`). `json.dumps()` is the stdlib canonical JSON serializer — the same pattern already used at lines ~402, ~415, ~594, ~607 in the same file.

### D3 — `import json` hoisted to module top
Reference: PEP 8 stdlib import ordering. Two local `import json` statements inside `load_rag_collections()` (~395) and `load_agents()` (~574) removed; single top-level import added (line 31). DRY win.

### D4 — Preventive maintenance casts at lines ~397/410/588/601 (WRITE_SET_EXTENSION)
Reference: task pack §"Other cast sites in the same file" (planner pragmatic option). Validator approved in `handoff §Validator review §Decisions`.
Rationale: All four lines have the same `:x::jsonb` cast pattern as the primary fix. Their tables (`rag_collections`, `agents`) don't exist after migration 0001, so these code paths are `deferred` in this slice's acceptance tests. Fixing them now prevents guaranteed defects when P02-S03 and P02-S07 activate those tables. Change is mechanical (cast syntax only; `json.dumps()` was already in place at those sites).
