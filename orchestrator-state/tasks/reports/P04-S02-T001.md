## Debug cycles summary

### Cycle 1 — Validator (cycle 0) findings F1 + F2

**F1 (CRITICAL)**: `canIndex = !!document.collection_id && isTerminal` predicate blocked Index CTA for status="uploaded" (the canonical post-upload state per backend service_upload.py:237). Admin could never trigger the first vectorization through the UI.

Fix: `canIndex = !!document.collection_id && !isInFlight`. `isTerminal` removed (no longer used). Regression tests T-DBG1..T-DBG6 added to RagDocumentsPage.test.tsx. Suite 205 → 211 PASS.

**F2 (HIGH)**: `indexLabel = status === "indexed" ? t("index.inProgress") : t("index")` was semantically inverted — an already-indexed (terminal) doc showed "Indexando". Fix: single label `indexLabel = t("documents.action.index")` always; in-progress progress communicated by aria-busy + status-cell TrackedLabel. T-DBG5 guards against locale-specific regression in all 3 locales.

### Cycle 2 — Screen/Journey reviewer finding TAP_TARGET_36PX

**TAP_TARGET_36PX (in-scope)**: "Ver colecciones →" button rendered at 36px height. Root cause: `COLLECTIONS_LINK_STYLE` had `display:"inline-block"` + `padding:0` + no `minHeight`.

Fix: `display:"inline-flex"` + `alignItems:"center"` + `minHeight:"44px"` in `RagDocumentsPage.styles.ts`. New companion test file `RagDocumentsPage.tap-target.test.tsx` added (existing page test file at 449 lines, over ~300 hard cap per rules/01-non-negotiables.md). T-DBG2-TAP1 asserts style-source contract; T-DBG2-TAP2 asserts rendered DOM inline style. Suite 211 → 213 PASS.
