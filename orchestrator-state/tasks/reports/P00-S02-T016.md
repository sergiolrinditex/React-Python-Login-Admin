## Decisions

| Decision | Rationale | Reference |
|---|---|---|
| Async `@pytest_asyncio.fixture` over sync variant | Sync fire-and-forget `loop.create_task(dispose())` (pattern in `test_auth_signup.py:99`) never awaits the dispose, leaving asyncpg connections alive on the old loop. Async fixture genuinely awaits `db_module._engine.dispose()` inside the test's own event loop, fully releasing connections before the next test. | Debugger handoff §fix_applied; TECHNICAL_GUIDE §6.5 (admin-ai) |
| `autouse=True, loop_scope="function"` at conftest level | Structural fix — any integration test that exercises a FastAPI handler touching DB can trigger the bug; opt-in would require per-file boilerplate. `autouse` in conftest.py DRYs the fix across all 9 sibling test files simultaneously. | Task pack §10 Plan A; validator round 2 §arquitectura |
| NOT modifying `test_auth_signup.py` | Its local sync `reset_db_engine_singleton` is in a protected write_set (P01-S02-T001 scope). Both fixtures are idempotent; the global async autouse runs first and already nulls the singletons, making the local sync version a no-op. DRY cleanup deferred to low-severity follow-up. | Handoff §Effective write set; task pack §17 R1 |
| `contextlib.suppress(Exception)` in teardown | Mirrors canonical helper in `test_auth_signup.py:117-125`. A broken pool from a failed test must never cascade into the next test's setup. The unconditional `db_module._engine = None` after suppress guarantees the next test starts clean even if dispose silently fails. | Debugger handoff §fix_applied |
