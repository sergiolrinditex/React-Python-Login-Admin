## Debugger cycle 1

**Finding F1** (validator cycle 1, blocking): `backend/app/users/deps.py:133` had `except (jwt.InvalidTokenError, Exception)` — broad `Exception` catch violates `01-non-negotiables.md` §Error handling.

**Root cause**: Developer caught bare `Exception` defensively to preserve the anti-enum 401 envelope across unexpected decode failures. Intent correct, typing wrong.

**Fix applied** (single line at deps.py:133):
```diff
-    except (jwt.InvalidTokenError, Exception):
+    except (jwt.InvalidTokenError, ValueError):
```

`ValueError` covers `decode_token`'s explicit purpose-mismatch raise. Every JWT decode failure inherits from `jwt.InvalidTokenError`. No behavior change — confirmed by 31/31 passing tests in both verbose modes after fix.

**Files touched by debugger**: only `backend/app/users/deps.py` (1 line replacement, no contract/schema/route/DB/audit changes).
