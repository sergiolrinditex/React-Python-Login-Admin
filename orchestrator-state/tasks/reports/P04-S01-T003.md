## Decisions

### Architecture

- **State machine as local hook** (`useState` + step enum + TanStack mutation/query): YAGNI over global store. TECHNICAL_GUIDE §6.1 `modelWizardStore` naming refers to local client-state slot, not a global redux/zustand mandate. ADR on `docs/source-of-truth/HILO_PEOPLE_TECHNICAL_GUIDE.md`.
- **Step extraction via `_` prefix siblings**: `_ModelWizardPage.<step>-step.tsx` pattern pre-authorized by task pack §7; avoids >300-line page file while keeping Clean Architecture (step components are prop-driven, zero business logic leak). Per validator memory note, future slices with similar multi-step pages should pre-create these sibling files instead of post-hoc splitting.
- **Helpers + types extraction from hook**: `useModelWizard.helpers.ts` (pure validators, no deps) + `useModelWizard.types.ts` (domain types + `classifySubmitError`). Re-exports from the hook keep public surface and test imports stable. Reference: `docs/source-of-truth/HILO_PEOPLE_TECHNICAL_GUIDE.md` §6.1.
- **`secret_plain` credential security — compensating controls**: `unique symbol` TS brand rolled back for cross-module TS compatibility; three-layer compensating control: `type="password"` input, hook-state-only (no persistence), `useEffect` cleanup + `onSuccess` zeroing. Public return strips `secret_plain` via destructure. PII-clean log test T26. Reference: `docs/source-of-truth/01-non-negotiables.md §security → claves de proveedores externos`.
- **i18n lockstep ES/EN/FR**: `modelsNew.*` keys added in lockstep across three locales; 7 test cases enforce key-parity. Reference: `docs/source-of-truth/STACK_PROFILE.yaml → frontend.i18n`.
