## Problem statement

React 18 StrictMode mounts `useEffect([])` twice in development. `AuthProvider.tsx:124` calls `repo.refresh()` from that effect. Without deduplication, two concurrent `POST /api/v1/auth/refresh` requests reach the backend. Because refresh token rotation is atomic (the first request revokes the old token and issues a new one), the second request arrives with an already-revoked token and receives a 401. Result: the user sees "session expired" on first paint after login, or opening `/chat` in a second tab invalidates the session of the first tab.
