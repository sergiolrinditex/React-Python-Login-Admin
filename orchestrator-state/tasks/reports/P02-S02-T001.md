## Decisions

| Decision | Detail | Source doc ref |
|----------|--------|----------------|
| D-ENC1: Lazy Fernet key validation | `lru_cache` on `_get_fernet()`; raises `EncryptionKeyError` with generation hint on placeholder `replace-with-dev-key` or missing key | TECHNICAL_GUIDE §10.2 |
| D-RL1: INCR + conditional EXPIRE atomicity | Redis pattern: `INCR key`; if return==1 → `EXPIRE key <window>`. Canonical race-safe Redis rate-limiter pattern. No Lua or pipeline needed for the first-hit branch | TECHNICAL_GUIDE §10.2; `redis.io` rate-limiting pattern |
| R2: ENCRYPTION_KEY separate from MFA_ENCRYPTION_KEY | Blast-radius isolation: AI/MCP credentials vs MFA TOTP secrets are different threat classes. `verification_data/crypto.py` keeps using `MFA_ENCRYPTION_KEY` unchanged | TECHNICAL_GUIDE §10.2; instrucciones §3.3 |
| R3: `auth/rate_limit.py` left unchanged | KISS scope: migrating 6 auth endpoints to Redis rate_limit in same slice would explode regression surface. Co-existence, future migration per-endpoint | instrucciones §3.3; TECHNICAL_GUIDE §10.2 |
| R4: `require_user` = re-export of `users.deps.get_current_user` | No JWT decode duplication; Clean Architecture — `security` delegates to `users.deps` (leaf) | TECHNICAL_GUIDE §5.1, §10.1, §10.2 |
