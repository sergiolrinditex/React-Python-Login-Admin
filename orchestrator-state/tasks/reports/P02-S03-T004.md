## Deliverables

### Scripts
- `scripts/gen-dev-secrets.sh` — NEW. Idempotent Fernet/JWT key provisioner. Rotates ENCRYPTION_KEY if placeholder (`replace-with-dev-key`) using `Fernet.generate_key()` inline via Python. Sets `chmod 600 .env`. POSIX portable (BSD awk + GNU awk). `set +x` prevents xtrace secret leak. Never prints key values — logs only `key_len=N`. Smoke-tests Fernet constructor validity.

### Backend — verification data module
- `backend/app/verification_data/loader_base.py` — NEW (83 LoC). Extracted shared primitives: `LoadResult`, `_table_exists`, `_info` verbose gate.
- `backend/app/verification_data/loader_ai_tables.py` — NEW (299 LoC). Two new loaders: `load_ai_provider_credentials()` (encrypts `credential_plain` at load time via `encrypt_secret`; upserts by `provider_id`), `load_ai_models()` (upserts by `provider_id + model_id`; preserves `is_default` invariant).
- `backend/app/verification_data/schemas.py` — EDIT (+85 LoC). New Pydantic fixtures: `AiProviderCredentialFixture` (provider_ref, auth_type, credential_plain, expires_at) and `AiModelFixture` (provider_ref, model_id, model_type, enabled, is_default, capabilities, pricing).
- `backend/app/verification_data/loader.py` — EDIT (617→575 LoC). Imports and re-exports new loaders from `loader_base.py` and `loader_ai_tables.py` for backward compatibility.
- `backend/app/verification_data/bootstrap.py` — EDIT. `_run_admin_ai_group()` now calls loaders in FK-safe order: providers → credentials → models. Wires new fixture subdirs `admin_ai/credentials` and `admin_ai/models`.
