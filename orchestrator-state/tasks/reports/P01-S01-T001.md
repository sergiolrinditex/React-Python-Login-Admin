## Architectural Decisions

| Decision | Source doc reference |
|----------|---------------------|
| `audit_logs` implements §10.3 verbatim (actor_user_id, action, entity_type, entity_id, metadata, created_at); ip/user_agent/request_id deferred to FU-20260509105319 | HILO_PEOPLE_TECHNICAL_GUIDE.md §10.3 wins over 01-non-negotiables.md §Audit log |
| JSONB imported from `sqlalchemy.dialects.postgresql`, not `sqlalchemy` | SQLAlchemy 2.x dialect-specific types (D2) |
| Python attribute `metadata_col` → DB column `"metadata"` via positional arg — avoids shadowing `Base.metadata` | SQLAlchemy declarative base collision (D3) |
| `_fresh_conn()` asynccontextmanager per test (new engine per test) — pytest-asyncio asyncio_mode=auto creates one event loop per test function | D4 event-loop isolation |
| Extensions NOT dropped on downgrade — other schemas may rely on them; CREATE IF NOT EXISTS is idempotent on re-upgrade | HILO_PEOPLE_TECHNICAL_GUIDE.md §10.3 §7.3 |
| Circular ORM references handled with `TYPE_CHECKING` guards + string-based forward refs | SQLAlchemy 2.x pattern |
| Single Alembic revision 0001 despite 422 lines — splitting would break Alembic revision tracking model | Alembic 1.18.4 single-revision discipline |
