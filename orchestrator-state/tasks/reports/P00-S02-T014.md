### New components / scripts
- **`scripts/setup-from-scratch.sh`**: Added `ensure_jwt_keypair()` function (~210 lines including embedded Python heredocs `PYVALIDATE` and `PYINJECT`) and invocation at line 148, immediately after `ensure_encryption_key`. The function:
  - Detects placeholder or invalid RSA keypair values in `.env` (empty, `<change-me>`, `dev-rsa-*-pem-placeholder`, parse error, or sign/verify mismatch between private and public key).
  - Generates RSA 2048 keypair using `cryptography==48.0.0` (`PrivateFormat.PKCS8`, `PublicFormat.SubjectPublicKeyInfo`, `NoEncryption()`), compatible with `pyjwt[crypto]` RS256.
  - Stores PEM inline in `.env` as double-quoted multi-line values (`JWT_PRIVATE_KEY` + `JWT_PUBLIC_KEY`).
  - Logs masked public-key thumbprint (last 4 hex chars of SHA-256(pub_pem)) — CWE-532 compliant, no PEM in stdout/stderr.
  - Idempotent: rerun on an already-valid keypair skips generation (sha256 of `.env` unchanged).
  - Graceful degradation: warns and returns 0 if python3 or `cryptography` package unavailable.
- **`.env.example`**: JWT block (lines 62-78) re-documented to explain auto-generation behavior, sentinel values (`dev-rsa-private-key-pem-placeholder`) preserved as detection triggers, "Do NOT replace by hand for dev" instruction added.
