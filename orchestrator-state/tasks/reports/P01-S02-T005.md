## Summary

Implemented the forgot-password and reset-password backend endpoints with production-grade security: anti-enumeration body+timing, one-use token enforcement via `SELECT FOR UPDATE`, Argon2 password rotation, session invalidation (all active refresh tokens revoked in a single transaction), D-S2 audit pattern, and a multi-locale (es/en/fr) mail outbox module. 21 integration tests pass in isolation; full suite shows 19 pre-existing environmental failures (DB schema pollution from migration downgrade test — not a regression from this slice).

## Key Decisions

| Decision | Source doc ref |
|----------|---------------|
| Anti-enumeration: always 200 + identical body, no side-effects for unknown email | OWASP Authentication Cheat Sheet §Forgot Password; TECHNICAL_GUIDE §2.6 H-forgot-2,6 |
| Token stored SHA-256 hashed (secrets.token_urlsafe raw transmitted, never stored) | TECHNICAL_GUIDE §2.6 H-forgot-5 |
| FOR UPDATE on token fetch to prevent concurrent reuse under race | TECHNICAL_GUIDE §2.6 H-reset-6 |
| All active refresh tokens revoked atomically on successful reset (single transaction) | TECHNICAL_GUIDE §2.6 H-reset-7 |
| Mail module with MAIL_MODE=dev writes JSONL outbox; no external call in dev/test | STACK_PROFILE.yaml; non-negotiable §Tests |
| Argon2id for password re-hash on reset (matching sign-up contract) | TECHNICAL_GUIDE §2.2 |
| D-S2 audit pattern: audit in same DB transaction as state mutation | TECHNICAL_GUIDE §4.2 D-S2 |
| Email PII in logs: email_domain only, never raw address or token | non-negotiables §Logging, §Security A09 |
| WRITE_SET_DRIFT: password_reset.py split → password_reset_request.py + password_reset_consume.py; mail/sender.py split → 4 files ≤200 LOC each | non-negotiables §File size; pre-declared in task pack §M.1 |
